# TOYBOX v2.0 バージョンアップ企画書

**作成日**: 2026年4月8日  
**作成者**: 開発チーム  
**対象サービス**: TOYBOX（https://toybox.ayatori-inc.co.jp）  
**運営**: AYATORI Inc.

---

## 1. 背景・現状認識

### サービス概要

TOYBOXは、AIを活用して画像・動画・ゲームを作成・共有するクリエイティブプラットフォームです。  
投稿するとカードや称号が報酬として付与されるゲーミフィケーション機能を備えており、  
StudySphereとのSSO連携により教育機関ユーザーも利用できます。

### 現状の課題

| 項目 | 現状 |
|------|------|
| アクティブユーザー | 約10名 |
| 主な課題 | コミュニティとしての盛り上がりに欠ける |
| 投稿への反応 | いいね（1種類）のみ |
| 称号システム | ランダム付与・1週間で失効・達成条件不明確 |
| ランキング | 当日のみ（継続動機が弱い） |

### 課題の本質

ユーザーが「**また来たい・投稿したい**」と感じる動機が不足しています。  
具体的には以下の3点に集約されます。

1. **投稿しても反応が少ない** → 投稿のやりがいがない
2. **毎日来る理由がない** → リピート率が上がらない
3. **成長・達成感が見えない** → 長期継続につながらない

---

## 2. 改善方針

> **「小さなコミュニティでも、一人ひとりが成長を実感できるプラットフォーム」**

メンタルに配慮したユーザー層に向けて、**否定や批判のない、ポジティブな交流と達成感**を設計の中心に置きます。コメント機能は意図的に実装せず、代わりに多様なポジティブリアクションで感情表現の幅を広げます。

---

## 3. 機能改善一覧

---

### 【改善①】ポジティブリアクションの拡充

#### 背景

現在のリアクションは「いいね（⭐）」1種類のみです。  
コメント機能はメンタルへの配慮から意図的に未実装としているため、  
リアクションのバリエーションを増やすことでコメントの代替とします。

#### 内容

以下6種類のリアクションを追加します。**すべてポジティブ表現のみ**とし、ネガティブな評価は一切含みません。

| アイコン | 名称 | 用途・場面 |
|---------|------|-----------|
| ⭐ | いいね | 基本・汎用（現状維持） |
| 🔥 | 熱い！ | 迫力ある・すごい動画・ゲーム |
| 😲 | おどろき | 技術力や完成度への驚き |
| 🥰 | かわいい | イラスト・キャラクター系 |
| 😂 | おもしろい | ユーモア・笑えるネタ作品 |
| 👏 | 拍手 | 力作・完成度の高い作品 |

#### 期待効果

- 投稿者が「どう受け取られたか」を具体的に知ることができる
- 閲覧者も「何も押せない」状況がなくなり、反応しやすくなる
- 数字だけでなく「種類」でフィードバックが得られるためモチベーションが上がる

### 【改善②】称号システムの完全刷新

#### 現状の問題点

| 問題 | 詳細 |
|------|------|
| ランダム付与 | 何をすれば何の称号が得られるか分からない |
| 有効期限あり | 1週間で称号が消える → 頑張りが無駄になる感覚 |
| 画像管理が大変 | 称号ごとに画像を用意する運用負荷が高い |
| コレクション感がない | 称号を集める楽しさがない |

#### 改善内容

**① 達成条件の明確化（アチーブメント制）**

「何をどれだけやったら、どの称号がもらえるか」を明示します。  
称号は条件を満たしたタイミングで自動付与されます。

**投稿数カテゴリ（例）**

| 投稿数 | 称号名 | 公開/非公開 |
|--------|--------|------------|
| 登録直後 | 駆け出しクリエイター | 公開（全員が最初に取得） |
| 5回 | 見習いクリエイター | 公開 |
| 20回 | 中堅クリエイター | 公開 |
| 50回 | ベテランクリエイター | 公開 |
| 100回 | ？？？ | 秘密（超高難易度） |

**ゲーム投稿カテゴリ（例）**

| 条件 | 称号名 | 公開/非公開 |
|------|--------|------------|
| ゲーム1本投稿 | ゲームメーカー見習い | 公開 |
| ゲーム5本投稿 | ゲームクリエイター | 公開 |
| ゲーム20本投稿 | ？？？ | 秘密 |

**反応・いいねカテゴリ（例）**

| 条件 | 称号名 |
|------|--------|
| いいね合計10件獲得 | 注目の新星 |
| いいね合計50件獲得 | 人気クリエイター |
| いいね合計200件獲得 | ？？？ |

**連続投稿カテゴリ（例）**

| 条件 | 称号名 |
|------|--------|
| 7日連続投稿 | 一週間の炎 |
| 30日連続投稿 | ？？？ |

---

**② 称号デザインの統一（SVGバッジ形式）**

- 現在の「画像ファイルごとに用意」方式をやめ、**統一フォーマットのSVGバッジ**に切り替えます
- デザインはTOYBOXのUIカラー（ダーク背景・ゴールドアクセント）に合わせた**バッジ型SVG**
- 難易度・カテゴリごとにカラーコードで視覚的に区別
- 合計**約50種類**を用意（初期実装分）

**カラー設計（案）**

| 難易度 | カラー | 対象 |
|--------|--------|------|
| 入門 | グリーン | 登録直後・初投稿 |
| 初級 | ブルー | 5〜20回投稿 |
| 中級 | ゴールド | 50回・連続投稿7日 |
| 上級 | レッド | 100回・連続30日 |
| 伝説 | 虹色（アニメーション） | 超高難易度・秘密称号 |

---

**③ 称号有効期限の廃止**

- **現行の「1週間で消える」仕様を完全廃止**
- 一度取得した称号は永続保有
- いつでも好きな称号に切り替えて表示可能（プロフィールに1つ表示）

---

**④ 称号コレクション画面の新設**

```
【称号コレクション画面イメージ】

━━━━━━━━━━━━━━━━━━━━━━━━━
  取得済み（点灯）        未取得（グレーアウト）
━━━━━━━━━━━━━━━━━━━━━━━━━

  [🟢 駆け出し]          [🔵 見習い（5回投稿で解放）]
  [🎮 ゲームメーカー]     [🔵 中堅（あと15回）]
  [💛 注目の新星]        [🟠 ??? （条件不明）]
                         [❓❓❓ （超秘密）]

━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 取得済み → **フルカラーで点灯**、クリックで「メインに設定」
- 未取得・条件公開 → **グレーアウト＋達成条件を表示**（例：「あと15回投稿で解放」）
- 未取得・秘密 → **「？？？」と表示**、達成すると初めて名前が明かされる
- 超高難易度 → **「❓❓❓」**で完全非公開、存在自体がミステリー

---

#### 期待効果

- 「次はあの称号を取りたい」という**継続投稿の目標**が生まれる
- 秘密称号・超高難易度称号が**コミュニティ内の話題**になる
- コレクション感により**長期定着率の向上**が見込める

### 【改善③】ポイントシステムの新設

#### 概要

投稿・ログイン・リアクション受け取り・ゲームプレイなどの行動に対してポイントを付与します。  
ポイントは**本人のみ閲覧可能**（他ユーザーには非公開）とし、現段階ではスコアとして蓄積するのみとします。  
将来的に AYATORI Inc. 独自ポイントとの連携を予定しています。

#### ポイント付与ルール

| アクション | ポイント | 条件・備考 |
|-----------|---------|-----------|
| 初回登録ボーナス | **100pt** | アカウント作成時に1回限り |
| 既存ユーザー移行ボーナス | **100pt** | ポイント機能リリース時に全ユーザーへ一括付与 |
| 毎日ログイン | **30pt** | 1日1回まで |
| 画像投稿 | **10pt** | 当日2件目以降は半額（5pt） |
| 動画投稿 | **25pt** | 当日2件目以降は半額（12pt） |
| ゲーム投稿 | **100pt** | 当日2件目以降は半額（50pt） |
| リアクションをもらう | **3〜5pt** | 1リアクションごと・種類による |
| ゲームを一定時間プレイされる | **20pt** | URLを開いただけはノーカウント・1プレイごと |

**リアクション種別ポイント内訳**

| リアクション | ポイント |
|------------|---------|
| ⭐ いいね | 3pt |
| 🔥 熱い・🥰 かわいい・😂 おもしろい | 4pt |
| 👏 拍手・😲 おどろき | 5pt |

#### 表示仕様

- **マイページ右上**に累計ポイント数を常時表示（本人のみ）
- ポイント獲得履歴（いつ・何で・何pt得たか）を本人が確認できる
- 他のユーザーのポイントは一切表示しない
- 管理者は管理画面から全ユーザーのポイントを確認可能

#### 将来展望

- AYATORI Inc. 独自ポイントとの連携（時期未定）
- ポイント交換・使用機能（時期未定）

#### 期待効果

- 毎日ログインする習慣が生まれる（ログインボーナス）
- 数字の積み上がりが視覚的に「成長実感」を与える
- 将来のポイント連携への期待感がサービスへの愛着につながる

### 【改善④】週間ランキングの導入

#### 現状の問題点

現在は「今日の投稿者ランキング」のみで、翌日にはリセットされます。  
10名という規模では当日のランキングは変動が少なく、競争・達成感が生まれにくい状態です。

#### 改善内容

**週間ランキング（月曜〜日曜）** を新設します。

| ランキング種別 | 集計内容 |
|--------------|---------|
| 週間投稿数ランキング | その週に一番多く投稿したユーザー |
| 週間いいね獲得ランキング | その週に最もいいねをもらった作品・ユーザー |

- マイページ・フィードページに常設表示
- 翌週月曜0時にリセット
- **TOP3には特別バッジ**（称号）を週替わりで付与（例：「今週の一番星」）

#### 期待効果

- 1週間という程よいスパンで「目標」と「達成感」が繰り返し生まれる
- 10名規模でも毎週順位が変動しやすく競争感が生まれる
- TOP3表彰による「みんなに認められる」体験

## 6. 技術的実現可能性

本企画の全機能は、**既存のアーキテクチャの範囲内で実装可能**です。

| 機能 | 実装方針 |
|------|---------|
| リアクション追加 | 既存 `Reaction.Type` にEnum値を追加するのみ |
| ポイントシステム | `UserMeta` または新規モデルにポイント残高・履歴を追加 |
| マイページポイント表示 | マイページ右上に累計ポイントを表示（本人のみ） |
| SVGバッジ | Djangoの `staticfiles` に配置、DB管理不要 |
| 称号アチーブメント | 投稿・いいね時のフックに条件チェックを追加 |
| コレクション画面 | 既存コレクションページを拡張 |
| 週間ランキング | 既存 `RankingDailyView` を週集計に拡張 |

追加のインフラ・外部サービスは不要です。

## 7. まとめ

本企画は「**機能を増やして複雑にする**」のではなく、  
**「今いる10名が毎日楽しめる・成長を感じられる仕組みを整える」** ことを最優先としています。

コメント機能のような諸刃の剣を避けつつ、  
ポジティブなリアクション・ポイント蓄積・明確な目標・コレクション達成感の4本柱で  
**少人数でも活気あるコミュニティ**を目指します。
