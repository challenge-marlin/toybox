## 問題の解決方法

1. **文字化けの問題**: UTF-8エンコーディングを明示的に設定
2. **画像未復元の問題**: メディアボリュームもリストア

## 完全リストア手順

### ステップ1: バックアップファイルをダウンロード

#### 1-1. ディレクトリを作成

```powershell
New-Item -ItemType Directory -Path "C:\backup\toybox\database" -Force
New-Item -ItemType Directory -Path "C:\backup\toybox\volumes" -Force
```

#### 1-2. SQLバックアップをダウンロード

```powershell
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144:/backup/toybox/database/toybox_*.sql.gz C:\backup\toybox\database\
```

#### 1-3. メディアボリュームをダウンロード

```powershell
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144:/backup/toybox/volumes/media_volume_*.tar.gz C:\backup\toybox\volumes\
```

---

### ステップ2: 現在のデータベースをバックアップ（念のため）

```powershell
cd C:\github\toybox\backend
docker compose exec -T db pg_dump -U postgres --encoding=UTF8 toybox > backup_before_restore_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql
```

---

### ステップ3: データベースを削除して再作成（UTF-8文字セット指定）

```powershell
# データベースを削除
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"

# UTF-8文字セットでデータベースを再作成
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"

# 文字セットを確認
docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
```

---

### ステップ4: SQLバックアップを解凍してリストア

#### 4-1. 最新のバックアップファイルを取得

```powershell
$backupFile = Get-ChildItem C:\backup\toybox\database\toybox_*.sql.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1
$backupFile.FullName
```

#### 4-2. 解凍してリストア（UTF-8エンコーディング設定）

**方法A: 7-Zipを使用（推奨）**

```powershell
# 7-Zipで解凍
$tempSqlFile = $backupFile.FullName -replace '\.gz$', ''
& "C:\Program Files\7-Zip\7z.exe" e $backupFile.FullName "-o$(Split-Path $tempSqlFile)" -y

# UTF-8エンコーディングでリストア
$content = Get-Content $tempSqlFile -Encoding UTF8 -Raw
$content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox

# 一時ファイルを削除
Remove-Item $tempSqlFile
```

**方法B: WSL/Git Bashを使用**

```powershell
# WSLで解凍してリストア
wsl gunzip -c "$($backupFile.FullName -replace '\\', '/' -replace 'C:', '/mnt/c')" | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```

---

### ステップ5: メディアボリューム（画像ファイル）をリストア

#### 5-1. 最新のメディアボリュームバックアップを取得

```powershell
$mediaBackup = Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1
$mediaBackup.FullName
```

#### 5-2. メディアボリュームをリストア

```powershell
# Dockerボリューム名を確認
docker volume ls | Select-String "media"

# メディアボリュームをリストア（ボリューム名を適宜変更）
docker run --rm `
  -v backend_media_volume:/data `
  -v "$(Split-Path $mediaBackup.FullName):/backup" `
  alpine sh -c "cd / && tar xzf /backup/$(Split-Path $mediaBackup.FullName -Leaf)"
```

**注意**: ボリューム名が `toybox_media_volume` の場合は、`backend_media_volume` を `toybox_media_volume` に変更してください。

### ステップ7: 復元の確認

```powershell
SELECT 
    'users' as table_name, COUNT(*) as count FROM users

SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'cards', COUNT(*) FROM gamification_card
UNION ALL
SELECT 'titles', COUNT(*) FROM gamification_title
UNION ALL
SELECT 'user_cards', COUNT(*) FROM gamification_usercard;
"@
```

#### 7-2. 日本語データが正しく表示されるか確認
