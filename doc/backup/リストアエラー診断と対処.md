# リストアエラー診断と対処ガイド

**作成日**: 2026年1月27日  
**目的**: リストア時に発生するエラーの診断と対処方法

---

## 現在のエラー状況

リストア時に以下のエラーが発生しています：

### エラー1: 文字エンコーディングの問題
```
ERROR: value too long for type character varying(200)
CONTEXT: COPY announcements, line 1, column title: "??????????OYBOX??v1.02..."
```
**原因**: バックアップファイルの文字エンコーディングが正しくない

### エラー2: 列の順序の問題
```
ERROR: invalid input syntax for type integer: "add_usercard"
CONTEXT: COPY auth_permission, line 25, column content_type_id: "add_usercard"

ERROR: invalid input syntax for type timestamp with time zone: ""
CONTEXT: COPY cards, line 1, column updated_at: ""

ERROR: invalid input syntax for type integer: "2026-01-23 00:34:35.298742+00"
CONTEXT: COPY titles, line 2, column duration_days: "2026-01-23 00:34:35.298742+00"

ERROR: invalid input syntax for type timestamp with time zone: "3"
CONTEXT: COPY user_meta, line 1, column updated_at: "3"
```
**原因**: バックアップファイルの列の順序が間違っている

### エラー3: JSON形式の問題
```
ERROR: invalid input syntax for type json
CONTEXT: JSON data, line 1: ["???", "????, "?...
COPY submissions, line 1, column steps: "["???", "????, "????]"
```
**原因**: JSONデータが文字化けしている

### エラー4: 外部キー制約違反
```
ERROR: insert or update on table "reactions" violates foreign key constraint
DETAIL: Key (submission_id)=(5) is not present in table "submissions".

ERROR: insert or update on table "user_cards" violates foreign key constraint
DETAIL: Key (card_id)=(5) is not present in table "cards".
```
**原因**: 参照先のテーブル（`submissions`, `cards`）のデータが正しくリストアされていない

## 対処方法

### 方法1: サーバー側でバックアップスクリプトを修正して新しいバックアップを取得（推奨）

**原因**: 現在のバックアップファイルは、修正前のスクリプトで作成された可能性があります。

**手順**:

1. **サーバー側でバックアップスクリプトが修正されているか確認**:
   ```bash
   # サーバー側で実行
   cat /var/www/toybox/scripts/backup_database.sh | grep -A 5 "pg_dump"
   ```
   
   **期待される出力**:
   ```bash
   docker exec backend-db-1 pg_dump \
       -U postgres \
       --encoding=UTF8 \
       --no-owner \
       --no-privileges \
       --clean \
       --if-exists \
       --verbose \
       toybox
   ```
   
   **修正されていない場合**: [バックアップスクリプト修正手順](./バックアップスクリプト修正手順.md)に従って修正してください。

2. **修正後のスクリプトで新しいバックアップを作成**:
   ```bash
   # サーバー側で実行
   /var/www/toybox/scripts/backup_database.sh
   
   # ログを確認
   tail -50 /var/log/toybox_backup.log
   
   # 新しいバックアップファイルを確認
   ls -lh /backup/toybox/database/toybox_*.sql.gz | tail -1
   ```

3. **新しいバックアップファイルをローカルにダウンロード**:
   ```powershell
   # PowerShellで実行
   # 最新のバックアップファイルをダウンロード
   scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
     root@160.251.168.144:/backup/toybox/database/toybox_YYYYMMDD_HHMMSS.sql.gz `
     C:\backup\toybox-verification\database\
   ```

4. **新しいバックアップファイルから再リストア**:
   - [バックアップ検証手順書](./バックアップ検証手順書.md)のステップ4を参照

### 方法2: 現在のバックアップファイルで可能な限りデータを復元

一部のテーブルは正しくリストアされているため、現在の状態でデータを確認できます。

```powershell
# 現在リストアされているデータを確認
docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
ORDER BY table_name;
"@

# サンプルデータを確認
Write-Host "`n=== カードデータのサンプル ===" -ForegroundColor Cyan
docker compose exec db psql -U postgres -d toybox -c "SELECT code, name, rarity FROM cards LIMIT 5;"

Write-Host "`n=== 投稿データのサンプル ===" -ForegroundColor Cyan
docker compose exec db psql -U postgres -d toybox -c "SELECT id, submitter_id, aim FROM submissions LIMIT 5;"
```

**注意**: この方法では、以下のテーブルが0件のままです：
- `announcements`
- `reactions`
- `titles`
- `user_meta`

これらは、サーバー側でバックアップスクリプトを修正して新しいバックアップを取得する必要があります。

## まとめ

現在のエラーは、**バックアップファイル自体に問題がある**ことが原因です。サーバー側のバックアップスクリプトを修正して、新しいバックアップファイルを取得することで解決できます。

**重要**: 修正前のバックアップスクリプトで作成されたバックアップファイルは、列の順序や文字エンコーディングの問題があるため、完全にリストアできません。必ず修正後のスクリプトで新しいバックアップを取得してください。
