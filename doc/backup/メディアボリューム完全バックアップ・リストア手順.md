# メディアボリューム完全バックアップ・リストア手順

## 概要

本番環境のメディアボリューム（画像ファイルなど）を完全バックアップし、ローカル環境でリストアして検証します。

## メディアボリュームの構成

- **本番環境**: `backend_media_volume` → `/app/public/uploads`
- **ローカル環境**: `media_volume` → `/app/public/uploads`

## 前提条件

- 本番サーバーへのSSHアクセス権限
- ローカル環境でDockerが起動している
- 十分なディスク容量（メディアファイルのサイズによる）

---

## ステップ1: 本番環境でメディアボリュームをバックアップ

### 1.1 本番サーバーにSSH接続

```bash
ssh user@production-server
```

### 1.2 メディアボリュームのバックアップを実行

```bash
# バックアップ日時
DATE=$(date +%Y%m%d_%H%M%S)

# バックアップディレクトリを作成
sudo mkdir -p /backup/toybox/media

# メディアボリュームをtar.gzでバックアップ
sudo docker run --rm \
  -v backend_media_volume:/source:ro \
  -v /backup/toybox/media:/backup \
  alpine tar czf /backup/media_volume_${DATE}.tar.gz -C /source .

# バックアップファイルのサイズと存在確認
ls -lh /backup/toybox/media/media_volume_${DATE}.tar.gz

# バックアップファイルの内容確認
sudo tar tzf /backup/toybox/media/media_volume_${DATE}.tar.gz | head -20
```

### 1.3 バックアップ情報の記録

```bash
# ファイル数を記録
echo "バックアップ日時: ${DATE}" > /backup/toybox/media/media_backup_${DATE}_info.txt
echo "元のボリューム: backend_media_volume" >> /backup/toybox/media/media_backup_${DATE}_info.txt
sudo tar tzf /backup/toybox/media/media_volume_${DATE}.tar.gz | wc -l >> /backup/toybox/media/media_backup_${DATE}_info.txt

# ハッシュ値を計算（整合性確認用）
sha256sum /backup/toybox/media/media_volume_${DATE}.tar.gz > /backup/toybox/media/media_volume_${DATE}.tar.gz.sha256
```

## ステップ2: バックアップファイルをローカルにダウンロード

### 2.1 ダウンロードディレクトリを作成

```powershell
# PowerShellでローカル環境にて実行
mkdir -p C:\github\toybox\backend\media_backups
cd C:\github\toybox\backend\media_backups
```

### 2.2 SCPでダウンロード

```powershell
# バックアップファイルをダウンロード（日時を実際の値に置き換え）
$DATE = "20260129_XXXXXX"  # 実際のタイムスタンプに置き換え

scp user@production-server:/backup/toybox/media/media_volume_${DATE}.tar.gz ./
scp user@production-server:/backup/toybox/media/media_volume_${DATE}.tar.gz.sha256 ./
scp user@production-server:/backup/toybox/media/media_backup_${DATE}_info.txt ./
```

### 2.3 ダウンロードファイルの整合性確認

```powershell
# PowerShellでSHA256確認
$expectedHash = (Get-Content "media_volume_${DATE}.tar.gz.sha256").Split(" ")[0]
$actualHash = (Get-FileHash "media_volume_${DATE}.tar.gz" -Algorithm SHA256).Hash.ToLower()

if ($expectedHash -eq $actualHash) {
    Write-Host "✅ ハッシュ値が一致しました" -ForegroundColor Green
} else {
    Write-Host "❌ ハッシュ値が一致しません！" -ForegroundColor Red
}
```

## ステップ3: ローカル環境でリストア

### 3.1 リストア前の現状確認

```powershell
# ローカル環境のメディアファイル数を確認
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -type f | wc -l"

# 現在のメディアボリュームをバックアップ（念のため）
$BACKUP_DATE = Get-Date -Format "yyyyMMdd_HHmmss"
docker run --rm `
  -v backend_media_volume:/source:ro `
  -v C:/github/toybox/backend/media_backups:/backup `
  alpine tar czf /backup/local_media_before_restore_${BACKUP_DATE}.tar.gz -C /source .
```

### 3.2 メディアボリュームをクリア

```powershell
# 既存のメディアファイルを削除
docker run --rm -v backend_media_volume:/data alpine sh -c "rm -rf /data/*"

# クリアされたことを確認
docker run --rm -v backend_media_volume:/data alpine sh -c "ls -la /data"
```

### 3.3 バックアップからリストア

```powershell
# バックアップファイルのパスを設定（実際のタイムスタンプに置き換え）
$DATE = "20260129_XXXXXX"
$BACKUP_FILE = "C:\github\toybox\backend\media_backups\media_volume_${DATE}.tar.gz"

# リストア実行
docker run --rm `
  -v backend_media_volume:/target `
  -v ${BACKUP_FILE}:/backup.tar.gz:ro `
  alpine sh -c "cd /target && tar xzf /backup.tar.gz"

Write-Host "✅ リストアが完了しました" -ForegroundColor Green
```

## ステップ4: リストア結果の検証

### 4.1 ファイル数の確認

```powershell
# リストア後のファイル数を確認
$restoredCount = docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -type f | wc -l"
Write-Host "リストアされたファイル数: $restoredCount" -ForegroundColor Cyan

# バックアップ情報ファイルから元のファイル数を確認
$originalCount = (Get-Content "media_backup_${DATE}_info.txt" | Select-Object -Last 1).Trim()
Write-Host "元のファイル数: $originalCount" -ForegroundColor Cyan

if ($restoredCount -eq $originalCount) {
    Write-Host "✅ ファイル数が一致しました" -ForegroundColor Green
} else {
    Write-Host "⚠️ ファイル数が異なります" -ForegroundColor Yellow
}
```

### 4.2 ディレクトリ構造の確認

```powershell
# ディレクトリ構造を確認
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -type d | sort"

# サブディレクトリごとのファイル数
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -type d -exec sh -c 'echo ""{}:""; ls -1 ""{}""| wc -l' \;"
```

### 4.3 サンプルファイルの確認

```powershell
# ランダムにファイルをサンプリング
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -type f | head -10"

# 特定の画像ファイルの存在確認（例）
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -name '*.jpg' | head -5"
docker run --rm -v backend_media_volume:/data alpine sh -c "find /data -name '*.png' | head -5"
```

### 4.4 ファイルサイズの確認

```powershell
# 総ファイルサイズを確認
docker run --rm -v backend_media_volume:/data alpine sh -c "du -sh /data"

# サブディレクトリごとのサイズ
docker run --rm -v backend_media_volume:/data alpine sh -c "du -sh /data/*"
```

### 4.5 ファイルパーミッションの確認

```powershell
# パーミッションを確認
docker run --rm -v backend_media_volume:/data alpine sh -c "ls -la /data | head -20"
```

### 5.1 開発サーバーを起動

```powershell
cd C:\github\toybox\backend

```

### 5.2 メディアファイルへのアクセステスト

ブラウザで以下を確認：

1. **ユーザープロファイル画像**: 
   - ログインしてプロファイル画像が表示されるか

2. **カード画像**: 
   - ガチャや所有カード一覧で画像が表示されるか

3. **その他のアップロードファイル**:

ブラウザの開発者ツール（F12）で：
- ネットワークタブで画像リクエストが404エラーにならないか確認
- コンソールで画像読み込みエラーがないか確認

## ステップ6: データベース内のメディアパスとの整合性確認

### 6.1 データベースに記録されているメディアパスを確認

```powershell
# PostgreSQLコンテナに接続
docker compose exec db psql -U postgres -d toybox

# ユーザープロファイル画像のパスを確認
SELECT id, username, email, 
       (SELECT profile_image FROM users_usermeta WHERE user_id = users_user.id) as profile_image
FROM users_user 
WHERE (SELECT profile_image FROM users_usermeta WHERE user_id = users_user.id) IS NOT NULL
LIMIT 10;

# カード画像のパスを確認
SELECT id, name, image, image_url 
FROM gamification_card 
WHERE image IS NOT NULL OR image_url IS NOT NULL
LIMIT 10;

\q
```

### 6.2 データベースのパスに対応するファイルが存在するか確認

```powershell
# データベースから取得したパスを使用（例: uploads/profiles/user_123.jpg）
$dbPath = "uploads/profiles/user_123.jpg"  # 実際のパスに置き換え

# ファイルの存在確認
docker run --rm -v backend_media_volume:/data alpine sh -c "ls -lh /data/${dbPath}"
```

## トラブルシューティング

### ケース1: ファイル数が一致しない

```powershell
# バックアップファイルの内容を確認
docker run --rm `
  -v ${BACKUP_FILE}:/backup.tar.gz:ro `
  alpine tar tzf /backup.tar.gz | wc -l

# 現在のボリュームのファイル数を再確認
docker run --rm -v backend_media_volume:/data alpine find /data -type f | Measure-Object | Select-Object -ExpandProperty Count
```

### ケース2: 画像が表示されない

1. **URLパスの確認**
   - ブラウザの開発者ツールで実際のリクエストURLを確認
   - Django設定の`MEDIA_URL`と`MEDIA_ROOT`を確認

2. **ファイルパーミッションの修正**
   ```powershell
   docker run --rm -v backend_media_volume:/data alpine chmod -R 755 /data
   ```

3. **Dockerコンテナの再起動**
   ```powershell
   docker compose restart web
   ```

### ケース3: バックアップファイルが大きすぎてダウンロードできない

```powershell
# 圧縮率を上げる（時間はかかるが容量削減）
docker run --rm \
  -v backend_media_volume:/source:ro \
  -v /backup/toybox/media:/backup \
  alpine tar cJf /backup/media_volume_${DATE}.tar.xz -C /source .

# または、分割ダウンロード
split -b 500M media_volume_${DATE}.tar.gz media_volume_${DATE}.tar.gz.part_
```
