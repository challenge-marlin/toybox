# バックアップスクリプト修正手順

**作成日**: 2026年1月27日  
**目的**: サーバー側のバックアップスクリプトを修正して、完全なデータバックアップを取得できるようにする

---

## 問題点

現在のバックアップスクリプト（`/var/www/toybox/scripts/backup_database.sh`）では、`pg_dump`コマンドにオプションが指定されていないため、以下の問題が発生していました：

1. **文字エンコーディングが明示されていない** → 文字化けが発生
2. **Dockerコンテナ内の環境変数が設定されていない** → `PGCLIENTENCODING`や`LANG`が未設定で文字化けが発生
3. **列の順序が正しくない可能性** → データ型エラーが発生
4. **オーナー情報や権限情報が含まれている** → 復元時のエラーが発生

---

## 修正内容

### 修正前

```bash
docker exec backend-db-1 pg_dump -U postgres toybox | gzip > $BACKUP_DIR/toybox_$DATE.sql.gz
```

### 修正後

```bash
# 環境変数を明示的に設定してUTF-8を保証
# PGCLIENTENCODING: PostgreSQLクライアントのエンコーディング
# LANG: システムのロケール設定
docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 backend-db-1 pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --verbose \
    --column-inserts \
    toybox 2>> $LOGFILE | gzip > $BACKUP_DIR/toybox_$DATE.sql.gz
```

### 追加したオプションの説明

| オプション | 説明 |
|-----------|------|
| `--encoding=UTF8` | 文字エンコーディングをUTF-8に明示的に指定（文字化けを防ぐ） |
| `--no-owner` | オーナー情報を除外（復元時のエラーを避ける） |
| `--no-privileges` | 権限情報を除外（復元時のエラーを避ける） |
| `--clean` | 既存のオブジェクトを削除するコマンドを含める |
| `--if-exists` | DROP文にIF EXISTSを追加（エラーを避ける） |
| `--verbose` | 詳細なログ出力（デバッグ用） |
| `--column-inserts` | **重要**: INSERT文で列名を明示的に指定（列の順序の問題を完全に回避） |
| `-e PGCLIENTENCODING=UTF8` | **重要**: Dockerコンテナ内のPostgreSQLクライアントエンコーディングをUTF-8に設定（文字化けを防ぐ） |
| `-e LANG=C.UTF-8` | **重要**: Dockerコンテナ内のシステムロケールをUTF-8に設定（文字化けを防ぐ） |
| 一時ファイル方式 | **重要**: パイプでgzipに渡すと文字エンコーディングが失われるため、一時ファイルに書き込んでからgzipする |
| Dockerコンテナ内で直接書き込み | **重要**: docker execの出力をホスト側にリダイレクトするとエンコーディングが失われるため、コンテナ内で直接ファイルに書き込んでからdocker cpでコピーする |

### 改善点

1. **`set -euo pipefail`**: エラー時に即座に終了、未定義変数の使用を防ぐ
2. **変数の明示的な定義**: `CONTAINER_NAME`, `DATABASE_NAME`, `DB_USER`を変数化
3. **事前チェック**: DockerコンテナとPostgreSQLの接続を事前に確認
4. **エラーハンドリング**: エラー処理関数を追加
5. **詳細な検証**: 主要テーブルのINSERT文を個別に確認
6. **ログ関数**: 統一されたログ出力形式

nanoで手動編集すると失敗する可能性があるため、以下の完全なスクリプトをコピー&ペーストしてください：

```bash
# バックアップスクリプトをバックアップ（念のため）
cp /var/www/toybox/scripts/backup_database.sh /var/www/toybox/scripts/backup_database.sh.backup

# 実行権限を確認・設定
chmod +x /var/www/toybox/scripts/backup_database.sh 2>/dev/null || true

# 完全なスクリプトを作成（改行コードは自動的にLFになる）
cat > /var/www/toybox/scripts/backup_database.sh << 'EOF'
#!/bin/bash
# TOYBOXデータベース自動バックアップスクリプト
# 作成日：2026年1月23日
# 更新日：2026年1月28日（完全なバックアップを取得するように完全修正）

set -euo pipefail  # エラー時に即座に終了、未定義変数の使用を防ぐ

# ============================================================================
# 設定
# ============================================================================

BACKUP_DIR="/backup/toybox/database"
DATE=$(date +%Y%m%d_%H%M%S)
LOGFILE="/var/log/toybox_backup.log"
NOTIFY_SCRIPT="/var/www/toybox/scripts/send_backup_notification.sh"
CONTAINER_NAME="backend-db-1"
DATABASE_NAME="toybox"
DB_USER="postgres"

# メール通知用の詳細情報を格納
DETAILS=""

# ============================================================================
# 関数定義
# ============================================================================

# ログ出力関数
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" >> "$LOGFILE"
}

# エラー処理関数
error_exit() {
    log "❌ ERROR: $1"
    DETAILS="$DETAILS\nエラー: $1"
    [ -x "$NOTIFY_SCRIPT" ] && $NOTIFY_SCRIPT "failure" "PostgreSQLダンプ" "$DETAILS" || true
    exit 1
}

# ============================================================================
# 初期化
# ============================================================================

log "======================================"
log "Backup started at $(date)"

# バックアップディレクトリが存在しない場合は作成
if [ ! -d "$BACKUP_DIR" ]; then
    mkdir -p "$BACKUP_DIR" || error_exit "バックアップディレクトリの作成に失敗しました: $BACKUP_DIR"
    log "バックアップディレクトリを作成しました: $BACKUP_DIR"
fi

# Dockerコンテナが起動しているか確認
if ! docker ps --format '{{.Names}}' | grep -q "^${CONTAINER_NAME}$"; then
    error_exit "Dockerコンテナが起動していません: $CONTAINER_NAME"
fi

# PostgreSQLが接続可能か確認
if ! docker exec "$CONTAINER_NAME" pg_isready -U "$DB_USER" > /dev/null 2>&1; then
    error_exit "PostgreSQLに接続できません: $CONTAINER_NAME"
fi

# ============================================================================
# バックアップ実行
# ============================================================================

BACKUP_FILE="${BACKUP_DIR}/toybox_${DATE}.sql.gz"
TEMP_SQL_FILE="${BACKUP_DIR}/toybox_${DATE}.sql"

log "バックアップファイル: $BACKUP_FILE"
log "一時SQLファイル: $TEMP_SQL_FILE"

# PostgreSQLダンプを実行
# オプション説明:
#   --encoding=UTF8          : 文字エンコーディングをUTF-8に明示的に指定（文字化けを防ぐ）
#   --no-owner               : オーナー情報を除外（復元時のエラーを避ける）
#   --no-privileges          : 権限情報を除外（復元時のエラーを避ける）
#   --clean                  : 既存のオブジェクトを削除するコマンドを含める
#   --if-exists              : DROP文にIF EXISTSを追加（エラーを避ける）
#   --verbose                : 詳細なログ出力
#   --column-inserts         : INSERT文で列名を明示的に指定（列の順序の問題を回避）
#                              → これにより、列の順序が変わっても正しくリストアできる
#   -e PGCLIENTENCODING=UTF8 : Dockerコンテナ内のPostgreSQLクライアントエンコーディングをUTF-8に設定
#   -e LANG=C.UTF-8          : Dockerコンテナ内のシステムロケールをUTF-8に設定
log "PostgreSQLダンプを実行中..."

# 環境変数を明示的に設定してUTF-8を保証
# パイプでgzipに渡すと文字エンコーディングが失われる可能性があるため、
# 一時ファイルに書き込んでからgzipする方法に変更
log "一時SQLファイルに書き込み中..."
if docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 "$CONTAINER_NAME" pg_dump \
    -U "$DB_USER" \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --verbose \
    --column-inserts \
    "$DATABASE_NAME" \
    2>> "$LOGFILE" > "$TEMP_SQL_FILE"; then
    
    # 一時ファイルをgzipで圧縮
    log "バックアップファイルを圧縮中..."
    if gzip -c "$TEMP_SQL_FILE" > "$BACKUP_FILE"; then
        # 一時ファイルを削除
        rm -f "$TEMP_SQL_FILE"
        log "✅ Backup successful: toybox_${DATE}.sql.gz"
    else
        error_exit "gzip圧縮に失敗しました"
    fi
else
    error_exit "pg_dumpの実行に失敗しました"
fi

# ============================================================================
# バックアップファイルの検証
# ============================================================================

# ファイルサイズを確認
if [ ! -f "$BACKUP_FILE" ]; then
    error_exit "バックアップファイルが作成されませんでした: $BACKUP_FILE"
fi

FILE_SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
FILE_SIZE_BYTES=$(stat -f%z "$BACKUP_FILE" 2>/dev/null || stat -c%s "$BACKUP_FILE" 2>/dev/null || echo "0")

log "File size: $FILE_SIZE ($FILE_SIZE_BYTES bytes)"
DETAILS="バックアップファイル: toybox_${DATE}.sql.gz\nファイルサイズ: $FILE_SIZE"

# ファイルサイズが0バイトの場合はエラー
if [ "$FILE_SIZE_BYTES" -eq 0 ]; then
    error_exit "バックアップファイルのサイズが0バイトです"
fi

# バックアップファイルが正常か簡易チェック（gzipの整合性）
if ! gunzip -t "$BACKUP_FILE" 2>> "$LOGFILE"; then
    error_exit "バックアップファイルが破損しています（gzip整合性チェック失敗）"
fi

log "✅ Backup file integrity OK (gzip check)"

# バックアップファイルの内容を検証
# --column-insertsオプションを使用しているため、INSERT文が生成される
log "バックアップファイルの内容を検証中..."

# INSERT文の数を確認
INSERT_COUNT=$(gunzip -c "$BACKUP_FILE" 2>/dev/null | grep -c "^INSERT INTO " 2>/dev/null || echo "0")
# 改行や空白を削除して数値に変換
INSERT_COUNT=$(echo "$INSERT_COUNT" | tr -d '\n\r ' || echo "0")
# 空文字列の場合は0にする
INSERT_COUNT=${INSERT_COUNT:-0}
log "INSERT statements found: $INSERT_COUNT"

# COPY文の数も確認（念のため）
COPY_COUNT=$(gunzip -c "$BACKUP_FILE" 2>/dev/null | grep -c "^COPY " 2>/dev/null || echo "0")
# 改行や空白を削除して数値に変換
COPY_COUNT=$(echo "$COPY_COUNT" | tr -d '\n\r ' || echo "0")
# 空文字列の場合は0にする
COPY_COUNT=${COPY_COUNT:-0}
log "COPY statements found: $COPY_COUNT"

# 総文数（数値として確実に計算）
TOTAL_STATEMENTS=$((INSERT_COUNT + COPY_COUNT))
DETAILS="$DETAILS\n整合性チェック: OK\nINSERT文の数: ${INSERT_COUNT}\nCOPY文の数: ${COPY_COUNT}"

# 文が少なすぎる場合は警告
if [ "$TOTAL_STATEMENTS" -lt 10 ]; then
    log "⚠️ Warning: Statement count is low ($TOTAL_STATEMENTS). Backup might be incomplete."
    DETAILS="$DETAILS\n警告: 文の数が少ない可能性があります（${TOTAL_STATEMENTS}件）"
fi

# 主要テーブルのINSERT文を確認
log "主要テーブルのINSERT文を確認中..."
for TABLE in users cards submissions user_cards announcements titles user_meta; do
    TABLE_COUNT=$(gunzip -c "$BACKUP_FILE" 2>/dev/null | grep -c "INSERT INTO.*${TABLE}" 2>/dev/null || echo "0")
    # 改行や空白を削除して数値に変換
    TABLE_COUNT=$(echo "$TABLE_COUNT" | tr -d '\n\r ' || echo "0")
    # 空文字列の場合は0にする
    TABLE_COUNT=${TABLE_COUNT:-0}
    if [ "$TABLE_COUNT" -gt 0 ]; then
        log "  ${TABLE}: ${TABLE_COUNT}件のINSERT文が見つかりました"
    else
        log "  ⚠️ ${TABLE}: INSERT文が見つかりませんでした"
    fi
done

# ============================================================================
# 古いバックアップの削除
# ============================================================================

# 7日以上古いバックアップを削除
OLD_COUNT=$(find "$BACKUP_DIR" -name "toybox_*.sql.gz" -mtime +7 2>/dev/null | wc -l 2>/dev/null || echo "0")
# 改行や空白を削除して数値に変換
OLD_COUNT=$(echo "$OLD_COUNT" | tr -d '\n\r ' || echo "0")
# 空文字列の場合は0にする
OLD_COUNT=${OLD_COUNT:-0}
if [ "$OLD_COUNT" -gt 0 ]; then
    log "Removing $OLD_COUNT old backup(s)"
    find "$BACKUP_DIR" -name "toybox_*.sql.gz" -mtime +7 -delete 2>/dev/null || true
    DETAILS="$DETAILS\n削除した古いバックアップ: ${OLD_COUNT}個"
fi

# 現在のバックアップ数を表示
BACKUP_COUNT=$(ls -1 "$BACKUP_DIR"/toybox_*.sql.gz 2>/dev/null | wc -l 2>/dev/null || echo "0")
# 改行や空白を削除して数値に変換
BACKUP_COUNT=$(echo "$BACKUP_COUNT" | tr -d '\n\r ' || echo "0")
# 空文字列の場合は0にする
BACKUP_COUNT=${BACKUP_COUNT:-0}
log "Total backups: $BACKUP_COUNT"
DETAILS="$DETAILS\n保持しているバックアップ: ${BACKUP_COUNT}個"

# ============================================================================
# 完了
# ============================================================================

log "Backup completed at $(date)"
log "======================================"

# メール通知（成功）
[ -x "$NOTIFY_SCRIPT" ] && $NOTIFY_SCRIPT "success" "PostgreSQLダンプ" "$DETAILS" || true

exit 0
EOF

# 実行権限を付与
chmod +x /var/www/toybox/scripts/backup_database.sh

# 改行コードを確認（LFであることを確認）
echo "改行コードを確認中..."
head -1 /var/www/toybox/scripts/backup_database.sh | od -c | head -1
# 出力例: "0000000   #   !   /   b   i   n   /   b   a   s   h  \n"
# \r（キャリッジリターン）が含まれていないことを確認

# スクリプトの構文を確認
bash -n /var/www/toybox/scripts/backup_database.sh
if [ $? -eq 0 ]; then
    echo "✅ スクリプトの構文チェック: OK"
else
    echo "❌ スクリプトの構文エラーがあります"
    exit 1
fi

# スクリプトの内容を確認（最初の10行）
head -10 /var/www/toybox/scripts/backup_database.sh
```

**重要**: 上記のコマンドをそのままコピー&ペーストして実行してください。`cat > ... << 'EOF'`の形式を使用することで、スクリプト全体を一度に作成できます。

## トラブルシューティング

### 問題1: 「pg_dump: error: unrecognized option '--if-exists'」

**原因**: PostgreSQLのバージョンが古い（9.5未満）

**解決方法**: `--if-exists`オプションを削除してください：

```bash
docker exec backend-db-1 pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --verbose \
    toybox 2>> $LOGFILE | gzip > $BACKUP_DIR/toybox_$DATE.sql.gz
```

### 問題2: 「COPY statements found: 0」

**原因**: バックアップファイルにデータが含まれていない

**解決方法**:
1. データベースにデータが存在するか確認：
   ```bash
   docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT COUNT(*) FROM users;"
   ```
2. `pg_dump`コマンドを直接実行してエラーを確認：
   ```bash
   docker exec backend-db-1 pg_dump -U postgres --encoding=UTF8 toybox | head -100
   ```

### 問題3: 「cannot execute: required file not found」

**原因**: 
- ファイルの改行コードがWindows形式（CRLF）になっている
- ファイルの実行権限が設定されていない
- シェバン行（`#!/bin/bash`）が正しく認識されていない

**解決方法**:
1. 改行コードをLFに変換：
   ```bash
   # dos2unixがインストールされている場合
   dos2unix /var/www/toybox/scripts/backup_database.sh
   
   # または、sedを使用
   sed -i 's/\r$//' /var/www/toybox/scripts/backup_database.sh
   ```

2. 実行権限を設定：
   ```bash
   chmod +x /var/www/toybox/scripts/backup_database.sh
   ```
