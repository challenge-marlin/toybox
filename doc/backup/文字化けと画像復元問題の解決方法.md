# 文字化けと画像復元問題の解決方法

**作成日**: 2026年1月29日  
**目的**: ホスグレのSQLデータをローカル環境にリストアした際の文字化けと画像未復元の問題を解決する

---

## 問題の概要

ホスグレのSQLデータをローカル環境にリストアすると、以下の2つの問題が発生します：

1. **日本語が「???????」と表示される（文字化け）**
2. **カードと称号の画像が復元されていない**

---

## 原因分析

### 1. 文字化けの原因

- **データベースの文字セットがUTF-8に設定されていない**
  - PostgreSQLのデフォルト文字セットが異なる可能性がある
  - データベース作成時にUTF-8を明示的に指定していない

- **リストア時のエンコーディング設定が不足している**
  - `pg_restore`や`psql`コマンド実行時にUTF-8エンコーディングが設定されていない
  - Dockerコンテナ内の環境変数（`PGCLIENTENCODING`、`LANG`）が設定されていない

### 2. 画像未復元の原因

- **画像ファイルはDockerボリュームに保存されている**
  - `docker-compose.yml`で`media_volume:/app/public/uploads`が定義されている
  - SQLデータベースには画像ファイルのパス情報のみが保存されている
  - 画像ファイル自体は別のボリュームに保存されているため、SQLデータだけでは復元されない

---

## 解決方法

### 解決方法1: データベース作成時にUTF-8を明示的に指定

```powershell
# PowerShellでの実行例
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

```bash
# Bashでの実行例
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

**重要なポイント**:
- `ENCODING='UTF8'`: データベースの文字エンコーディングをUTF-8に設定
- `LC_COLLATE='C'`と`LC_CTYPE='C'`: ロケール設定をC（ASCII互換）に設定
- `TEMPLATE=template0`: デフォルトテンプレートではなく、空のテンプレートを使用

### 解決方法2: リストア時にUTF-8エンコーディングを設定

#### PowerShellでのリストア

```powershell
# SQLファイルをUTF-8エンコーディングで読み込んでリストア
$content = Get-Content backup_file.sql -Encoding UTF8 -Raw
$content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```

**重要なポイント**:
- `-Encoding UTF8`: PowerShellの`Get-Content`でUTF-8エンコーディングを明示的に指定
- `-e PGCLIENTENCODING=UTF8`: Dockerコンテナ内のPostgreSQLクライアントエンコーディングをUTF-8に設定
- `-e LANG=C.UTF-8`: Dockerコンテナ内のシステムロケールをUTF-8に設定

#### Bashでのリストア

```bash
# gunzipで解凍しながらUTF-8エンコーディングを設定してリストア
gunzip -c backup_file.sql.gz | docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -i backend-db-1 psql -U postgres toybox
```

### 解決方法3: メディアボリュームのリストア

画像ファイルを復元するには、メディアボリュームのバックアップもリストアする必要があります。

#### ステップ1: 本番サーバーからメディアボリュームのバックアップをダウンロード

```powershell
# PowerShellでのダウンロード例
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes/media_volume_*.tar.gz `
  C:\backup\toybox\volumes\
```

#### ステップ2: メディアボリュームをリストア

```powershell
# PowerShellでのリストア例
$mediaBackup = Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1

docker run --rm `
  -v toybox_media_volume:/data `
  -v "$(Split-Path $mediaBackup.FullName):/backup" `
  alpine sh -c "cd / && tar xzf /backup/$(Split-Path $mediaBackup.FullName -Leaf)"
```

**注意**: Dockerボリューム名が異なる場合は、以下のコマンドで確認してください：

```powershell
docker volume ls | Select-String "media"
```

## トラブルシューティング

### 問題1: 日本語が「???????」と表示される

**確認事項**:
1. データベースの文字セットがUTF-8か確認：
   ```powershell
   docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
   ```
   出力が`UTF8`であることを確認してください。

2. リストア時にUTF-8エンコーディングが設定されているか確認：
   - `PGCLIENTENCODING=UTF8`と`LANG=C.UTF-8`が設定されているか確認
   - PowerShellの`Get-Content`で`-Encoding UTF8`が指定されているか確認

**解決方法**:
1. データベースを再作成してUTF-8を明示的に指定（上記の解決方法1を参照）
2. UTF-8エンコーディングを設定して再リストア（上記の解決方法2を参照）

### 問題2: 画像が表示されない

**確認事項**:
1. メディアボリュームのバックアップが存在するか確認：
   ```powershell
   Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz
   ```

2. 画像ファイルが存在するか確認：
   ```powershell
   docker compose exec web ls -la /app/public/uploads/cards/
   docker compose exec web ls -la /app/public/uploads/titles/
   ```

3. ボリュームが正しくマウントされているか確認：
   ```powershell
   docker compose exec web mount | Select-String "uploads"
   ```

**解決方法**:
1. メディアボリュームのバックアップをダウンロード（上記の解決方法3を参照）
2. メディアボリュームをリストア（上記の解決方法3を参照）

### 問題3: リストア中にエラーが発生する

**確認事項**:
1. Dockerコンテナが起動しているか確認：
   ```powershell
   docker compose ps
   ```

2. PostgreSQLが接続可能か確認：
   ```powershell
   docker compose exec db pg_isready -U postgres
   ```

3. エラーログを確認：
   ```powershell
   docker compose logs db | Select-Object -Last 50
   ```

**解決方法**:
- エラーメッセージを確認して、適切な対処を行う
- データベースを再作成してから再リストアを試す

---

## まとめ

文字化けと画像未復元の問題を解決するには、以下の3つのポイントが重要です：

1. **データベース作成時にUTF-8を明示的に指定**
   ```sql
   CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;
   ```

2. **リストア時にUTF-8エンコーディングを設定**
   ```powershell
   $content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
   ```

3. **メディアボリュームもリストア**
   ```powershell
   docker run --rm -v toybox_media_volume:/data -v /backup:/backup alpine tar xzf /backup/media_volume_*.tar.gz -C /data
   ```

これらの手順に従うことで、文字化けと画像未復元の問題が解決されます。
