# サーバー側バックアップファイル確認手順

**作成日**: 2026年1月28日  
**目的**: サーバー側に実際に存在するバックアップファイルを確認する

---

## 確認手順

### ステップ1: バックアップディレクトリの存在確認

```bash
# サーバー側で実行
ls -ld /backup/toybox/database

# ディレクトリが存在しない場合は作成
mkdir -p /backup/toybox/database
```

### ステップ2: すべてのバックアップファイルを一覧表示

```bash
# すべてのバックアップファイルを確認
ls -lh /backup/toybox/database/toybox_*.sql.gz

# 日付順にソートして最新の5件を表示
ls -lht /backup/toybox/database/toybox_*.sql.gz | head -5
```

### ステップ3: 最新のバックアップファイルを特定

```bash
# 最新のバックアップファイルを取得
LATEST_BACKUP=$(ls -t /backup/toybox/database/toybox_*.sql.gz 2>/dev/null | head -1)

if [ -z "$LATEST_BACKUP" ]; then
    echo "❌ バックアップファイルが見つかりません"
    echo "バックアップスクリプトを実行してください:"
    echo "  /var/www/toybox/scripts/backup_database.sh"
else
    echo "最新のバックアップファイル: $LATEST_BACKUP"
    ls -lh "$LATEST_BACKUP"
fi
```

### ステップ4: バックアップスクリプトを手動で実行

バックアップファイルが存在しない場合は、手動でバックアップを作成してください：

```bash
# バックアップスクリプトを実行
/var/www/toybox/scripts/backup_database.sh

# 実行結果を確認
echo "Exit code: $?"

# ログを確認
tail -50 /var/log/toybox_backup.log

# 作成されたバックアップファイルを確認
ls -lht /backup/toybox/database/toybox_*.sql.gz | head -1
```

### ステップ5: バックアップファイルの内容を確認

```bash
# 最新のバックアップファイルを取得
LATEST_BACKUP=$(ls -t /backup/toybox/database/toybox_*.sql.gz | head -1)

if [ -n "$LATEST_BACKUP" ]; then
    echo "=== バックアップファイルの情報 ==="
    echo "ファイル: $LATEST_BACKUP"
    echo "サイズ: $(ls -lh "$LATEST_BACKUP" | awk '{print $5}')"
    echo ""
    
    echo "=== INSERT文の数 ==="
    INSERT_COUNT=$(gunzip -c "$LATEST_BACKUP" | grep -c "^INSERT INTO " 2>/dev/null || echo "0")
    echo "総INSERT文数: $INSERT_COUNT"
    echo ""
    
    echo "=== 主要テーブルのINSERT文の数 ==="
    for TABLE in users cards submissions user_cards announcements titles user_meta; do
        COUNT=$(gunzip -c "$LATEST_BACKUP" | grep -c "INSERT INTO.*${TABLE}" 2>/dev/null || echo "0")
        echo "${TABLE}: ${COUNT}件"
    done
else
    echo "❌ バックアップファイルが見つかりません"
fi
```

## トラブルシューティング

### 問題1: バックアップディレクトリが存在しない

```bash
# ディレクトリを作成
mkdir -p /backup/toybox/database

# 権限を確認
ls -ld /backup/toybox/database
```

### 問題2: バックアップスクリプトが実行できない

```bash
# スクリプトの存在確認
ls -l /var/www/toybox/scripts/backup_database.sh

# 実行権限を確認
test -x /var/www/toybox/scripts/backup_database.sh && echo "実行可能" || echo "実行権限なし"

# 実行権限を付与
chmod +x /var/www/toybox/scripts/backup_database.sh
```

### 問題3: Dockerコンテナが起動していない

```bash
# Dockerコンテナの状態を確認
docker ps | grep backend-db-1

# コンテナが起動していない場合は起動
cd /var/www/toybox/backend
docker compose ps
docker compose up -d db
```

### 問題4: PostgreSQLに接続できない

```bash
# PostgreSQLの接続確認
docker exec backend-db-1 pg_isready -U postgres

# データベースの存在確認
docker exec backend-db-1 psql -U postgres -l | grep toybox
```

## バックアップファイルが見つからない場合の対処

### 方法1: 手動でバックアップを作成

```bash
# 手動でバックアップを作成
cd /var/www/toybox/backend
docker compose exec -T db pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --verbose \
    --column-inserts \
    toybox | gzip > /backup/toybox/database/toybox_manual_$(date +%Y%m%d_%H%M%S).sql.gz

# 作成されたファイルを確認
ls -lh /backup/toybox/database/toybox_manual_*.sql.gz | tail -1
```

### 方法2: バックアップスクリプトを直接実行

```bash
# バックアップスクリプトを実行
bash /var/www/toybox/scripts/backup_database.sh

# エラーが出た場合はログを確認
tail -100 /var/log/toybox_backup.log
```
