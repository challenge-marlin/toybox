## 🔧 使用するスクリプト

### データベース関連

- `backend/restore_from_prod.ps1` - データベースリストアスクリプト（PowerShell）
- `backend/reset_database.ps1` - データベースリセットスクリプト
- `scripts/backup_database.sh` - データベースバックアップスクリプト（Bash）
- `scripts/restore_backup.sh` - データベースリストアスクリプト（Bash）

### メディアボリューム関連

- `scripts/backup_media_from_prod.sh` - メディアバックアップスクリプト（本番環境）
- `backend/restore_media_local.ps1` - メディアリストアスクリプト（ローカル環境）
- `backend/verify_media_restore.ps1` - メディアリストア検証スクリプト

### その他

- `scripts/backup_nightly.sh` - **毎日21:00想定**：Web停止 → DB`.dump` → メディア`.tar.gz` → 再開（完全バックアップ）。世代保持は環境変数 `BACKUP_KEEP_GENERATIONS`（既定 **1**＝最新のみ・前日分は削除）
- `scripts/backup_retention.sh` - 上記から読み込み（単体実行不要）
- `scripts/backup_volumes.sh` - メディアボリューム完全バックアップ（`media_volume_YYYYMMDD_HHMMSS.tar.gz`）
- `scripts/backup_volumes_incremental.sh` - 差分バックアップ（※現在は運用で使用しない／初心者ガイドには出さない）

## 📝 基本的な作業フロー

### データベースのバックアップ・リストア

```
本番環境 → バックアップ → ダウンロード → ローカルリストア → 検証
   ↓           ↓              ↓              ↓             ↓
[サーバー]  [SQLファイル]  [ローカル]     [DB復元]      [確認]
```

### メディアボリュームのバックアップ・リストア

```
本番環境 → バックアップ → ダウンロード → ローカルリストア → 検証
   ↓           ↓              ↓              ↓             ↓
[サーバー]  [tar.gz]      [ローカル]     [ボリューム]  [画像確認]
```

## ⚠️ 重要な注意事項

1. **バックアップ前の確認**
   - 十分なディスク容量があることを確認
   - 本番環境への影響を最小限にするため、低負荷時に実行

2. **リストア前の確認**
   - 現在のローカルデータをバックアップ
   - Dockerコンテナが起動していることを確認

3. **文字エンコーディング**
   - UTF-8を使用することを推奨
   - Windows環境では文字化けに注意

4. **セキュリティ**
   - バックアップファイルには機密情報が含まれるため、取り扱いに注意
   - `.env`ファイルは含めない
