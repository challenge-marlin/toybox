# TOYBOX! 仕様・利用説明書

本書は、リポジトリ上の実装に基づく **現行仕様の説明** です。運用ポリシーや将来の変更は、別途お知らせやコードを参照してください。

---

## 1. TOYBOX! とは

クリエイターが **画像・動画・ゲーム** を投稿し、コミュニティで **リアクション** や **称号** を楽しむための Web サービスです。  
Ver2.0 以降は **TOYBOXポイント（TP）** と **アチーブメント称号（獲得後は期限なし）** が中心のゲーミフィケーションになっています。

---

## 2. 主な画面（フロントエンド URL）

| パス | 内容 |
|------|------|
| `/` | トップ |
| `/me/` | マイページ（投稿・タイムライン・ランキング・TP など） |
| `/feed/` | みんなの投稿（フィード） |
| `/profile/` | プロフィール設定 |
| `/profile/view/` | 公開プロフィール（自分／他者） |
| `/titles/` | 獲得称号一覧・表示中称号の切り替え |
| `/collection/` | カードコレクション |
| `/lottery/` | 抽選（カード関連） |
| `/login/` `/signup/` | ログイン・新規登録 |
| `/announcements/` | お知らせ一覧 |
| `/terms/` など | 利用規約・プライバシー等 |

マイページでは **「AIでゲームをつくる」** など外部ツールへの導線が用意されている場合があります（URL は環境により異なります）。

---

## 3. アカウント・認証

- **メール＋パスワード** による登録・ログイン（JWT アクセストークン／リフレッシュ）。
- **Discord OAuth** によるログイン連携（`/api/auth/discord/` 系）。
- **StudySphere** 連携用のプロフィール API（トークン設定・リンク等）が別途存在します。

API の認証ヘッダ例: `Authorization: Bearer <access_token>`

---

## 4. 投稿

### 4.1 種別

| 種別 | 概要 |
|------|------|
| 画像 | 画像ファイルをアップロード |
| 動画 | 動画ファイルをアップロード |
| ゲーム | ZIP をアップロードし、展開後の `index.html` 等を **ゲーム URL** として公開 |

### 4.2 主な API（抜粋）

- 画像・動画アップロード: `/api/submit/upload`（末尾スラッシュ有無の両方に対応するルートあり）
- ゲーム ZIP アップロード: `/api/submit/uploadGame`
- 投稿確定: `/api/submit/`
- 単体取得: `/api/submissions/<id>/`
- 自分の提出一覧: `/api/user/submissions/`（マイページ・プロフィールで利用）
- 他者の提出: `/api/user/submissions/<display_id>/`

削除済み投稿はソフトデリート等の仕様に従います（詳細は `submissions` アプリのモデルを参照）。

## 5. フィード・タイムライン・ランキング

- **フィード**: `/api/feed/`（人気・ハッシュタグ等の補助エンドポイントあり）
- **タイムライン**: `/api/timeline/`（マイページ周辺で利用）
- **本日の投稿者**: `/api/submitters/today/`
- **日次ランキング**: `/api/ranking/daily/`

## 6. リアクション

### 6.1 種類（6 種）

| 内部キー | 表示ラベル（例） | 絵文字（表示用） |
|----------|------------------|------------------|
| `submit_medal` | いいね！ | 👍 |
| `awesome` | すごい！ | 🤩 |
| `cute` | かわいい！ | 🥰 |
| `funny` | 笑える！ | 😂 |
| `moved` | 感動した | 😭 |
| `cool` | かっこいい！ | 😎 |

同一ユーザー・同一投稿・同一種類は **1 種類につき 1 回**（トグルで解除可能）。  
API: `POST /api/submissions/<id>/react/<reaction_type>/` および後方互換の `submit_medal` 用ルート。

### 6.2 通知

他人からリアクションが付いた場合、投稿者の **通知一覧** に追加されます（自分へのリアクションは通知しない実装）。

## 7. TOYBOXポイント（TP）

TP は **`UserPoint.total_points`** に累積し、**`PointHistory`** に履歴が残ります。  
「1 日」の境界は、サーバー設定 **`Asia/Tokyo`** の日付で判定される処理が多いです（`timezone.now().date()` 等）。

### 7.1 獲得条件一覧（実装ベース）

| 条件 | TP | 備考 |
|------|-----|------|
| 新規登録（初回） | **+100** | 登録ボーナス。履歴で二重付与を防止 |
| 移行ボーナス（Ver2.0 等） | **+100** | ユーザーごと 1 回（`migration_bonus_granted` フラグ） |
| 毎日チェックイン | **+30** | `POST /api/users/daily-checkin/`。同一日 1 回。マイページ表示時にフロントから呼び出し |
| 画像を投稿（当日 1 件目） | **+10** | 同日 2 件目以降は **+5** |
| 動画を投稿（当日 1 件目） | **+25** | 同日 2 件目以降は **+12** |
| ゲームを投稿（当日 1 件目） | **+100** | 同日 2 件目以降は **+50** |
| 投稿者がリアクションを受け取る | **+3〜+5** | 種類ごとに `REACTION_POINTS` で定義（下表） |
| 他人の投稿にリアクションを付ける | **+1** | 投稿者本人への自分リアクションは対象外 |
| ゲームを「開いてから 30 秒経過」扱い | プレイヤー **+5**、投稿者 **+10** | 下記「ゲームプレイ TP」参照 |

#### リアクション受取（投稿者側）の内訳

| リアクション | TP |
|--------------|-----|
| いいね（submit_medal） | 3 |
| すごい（awesome） | 5 |
| かわいい（cute） | 4 |
| 笑える（funny） | 4 |
| 感動した（moved） | 4 |
| かっこいい（cool） | 5 |

### 7.2 ゲームプレイ TP（30 秒）

- ライトボックスで **「ゲームを開く」リンクをクリック** した時点から、フロント側で **30 秒のタイマー** を開始します。
- 30 秒経過後に `POST /api/submissions/<id>/game-played/` が呼ばれ、サーバーで付与判定を行います。
- **自分のゲーム投稿には TP 付与なし**（プレイヤー・投稿者とも）。
- **同一ゲーム・同一日**あたりの重複付与は、履歴の `action_type` と説明文・日付でサーバー側で抑制します。
- ライトボックスを閉じるとタイマーはキャンセルされます（再度開いてクリックし直す必要あり）。

### 7.3 ポイント確認 API

- `GET /api/users/points/` … 残高と直近履歴（最大 50 件程度の実装）

### 7.4 履歴の `action_type`（コード上で使われる値）

モデル定義に加え、実装上 **文字列で追加** されている種別があります（例: `reaction_given`, `game_played_player`, `game_played_author`）。  
UI のラベルは `me.html` 等の `ACTION_LABELS` と対応します。

## 8. 称号（アチーブメント）

- 定義は `gamification/services.py` の **`ACHIEVEMENT_DEFINITIONS`**（現状 **51 件**、うち公式専用 1 件）。
- 条件達成で **`UserMeta.earned_titles`** に称号名が追加され、**期限は設けない**方針（Ver2.0）。
- 表示中の称号は API で更新（`POST /api/users/title/` 等）。
- 一覧・条件文の取得: `GET /api/users/achievements/`
  - **秘密・超秘密** の称号は、未獲得時に条件や名前を伏せる挙動があります（`secret` / `ultra_secret`）。
  - **`TOYBOX!公式`** は特定 `display_id` のユーザーのみ API に露出し、他ユーザーには一覧に出ません。

### 8.1 公式アカウント表示

- `TOYBOX!公式` を保有しているユーザーは、プロフィール API 等で **`isOfficial: true`** が返る場合があり、アバター周りの装飾が付きます。

## 9. プロフィール

- 更新: `PATCH/PUT` 等 `/api/user/profile/`（実装参照）
- 他人閲覧: `GET /api/user/profile/<anon_id>/`（`display_id` ベース）
- アバター・ヘッダ画像: `/api/user/profile/upload/` 等

---

## 10. カード・抽選

- カード一覧・所持: `/api/cards/me/`、`/api/cards/summary/` 等（`gamification/urls.py`）
- 投稿時の即時報酬は **主にカード抽選**（称号のランダム付与は廃止し、アチーブメント側に寄せるコメントあり）
- 抽選: `/api/lottery/` 配下（`lottery` アプリ）

---

## 11. 通知・Discord・その他 API

- 通知: `GET /api/users/notifications/`、`POST /api/users/notifications/read/` 等
- Discord へのシェア: `/api/share/discord/` 系（画像・動画ライトボックスから利用する想定）
- お問い合わせ: `/api/inquiry/`、`/api/contact/` 等
- 管理者用 API: `/api/admin/...`（別途権限・管理 URL）
