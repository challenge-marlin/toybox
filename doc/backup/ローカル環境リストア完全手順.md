# ローカル環境への完全リストア手順

**作成日**: 2026年1月29日  
**目的**: ホスグレのSQLデータと画像ファイルをローカル環境に完全にリストアする

---

## 問題点

現在、ホスグレのSQLデータをローカル環境にリストアすると、以下の問題が発生しています：

1. **日本語が「???????」と表示される（文字化け）**
   - 原因: リストア時に文字エンコーディングが正しく設定されていない
   - 解決策: UTF-8エンコーディングを明示的に設定してリストア

2. **カードと称号の画像が復元されていない**
   - 原因: 画像ファイルはDockerボリューム（`media_volume`）に保存されているため、SQLデータだけでは復元されない
   - 解決策: ボリュームのバックアップもダウンロードしてリストアする必要がある

---

## 前提条件

- Docker Desktopが起動していること
- PostgreSQLとRedisコンテナが起動していること
- 本番サーバー（160.251.168.144）にアクセスできること

---

## 完全リストア手順

### ステップ1: 本番サーバーからバックアップファイルをダウンロード

#### 1-1. SQLデータベースバックアップのダウンロード

**方法A: SCPを使用（推奨）**

```powershell
# PowerShellで実行
# 本番サーバーから最新のSQLバックアップをダウンロード
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/database/toybox_*.sql.gz `
  C:\backup\toybox\database\

# または、最新のファイルを指定
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/database/toybox_20260129_*.sql.gz `
  C:\backup\toybox\database\
```

**方法B: WinSCPを使用**

1. WinSCPでサーバーに接続
2. `/backup/toybox/database/` から最新の `toybox_*.sql.gz` をダウンロード
3. `C:\backup\toybox\database\` に保存

#### 1-2. メディアボリュームバックアップのダウンロード

**方法A: SCPを使用（推奨）**

```powershell
# PowerShellで実行
# 本番サーバーからメディアボリュームのバックアップをダウンロード
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes/media_volume_*.tar.gz `
  C:\backup\toybox\volumes\
```

**方法B: WinSCPを使用**

1. WinSCPでサーバーに接続
2. `/backup/toybox/volumes/` から最新の `media_volume_*.tar.gz` をダウンロード
3. `C:\backup\toybox\volumes\` に保存

**注意**: メディアボリュームのバックアップが存在しない場合は、本番サーバーで手動バックアップを作成してください：

```bash
# 本番サーバーで実行
docker run --rm \
  -v backend_media_volume:/data \
  -v /backup/toybox/volumes:/backup \
  alpine tar czf /backup/media_volume_$(date +%Y%m%d).tar.gz /data
```

### ステップ2: SQLデータベースのリストア（文字化け対策付き）

#### 2-1. 現在のデータベースをバックアップ（念のため）

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# 現在のデータベースをバックアップ
$backupDate = Get-Date -Format 'yyyyMMdd_HHmmss'
docker compose exec -T db pg_dump -U postgres --encoding=UTF8 toybox > "backup_before_restore_${backupDate}.sql"
```

#### 2-2. データベースを削除して再作成（UTF-8文字セット指定）

```powershell
# データベースを削除
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"

# UTF-8文字セットを明示的に指定してデータベースを再作成
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

#### 2-3. SQLファイルを解凍してリストア（UTF-8エンコーディング設定）

```powershell
# 最新のバックアップファイルを取得
$backupFile = Get-ChildItem C:\backup\toybox\database\toybox_*.sql.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1

if (-not $backupFile) {
    Write-Host "エラー: バックアップファイルが見つかりません" -ForegroundColor Red
    exit 1
}

Write-Host "バックアップファイル: $($backupFile.FullName)" -ForegroundColor Green

# 一時的に解凍したSQLファイルを作成
$tempSqlFile = $backupFile.FullName -replace '\.gz$', ''
Write-Host "SQLファイルを解凍中..." -ForegroundColor Yellow

# 7-ZipまたはPowerShellのExpand-Archiveを使用して解凍
# 7-Zipがインストールされている場合
& "C:\Program Files\7-Zip\7z.exe" e $backupFile.FullName "-o$(Split-Path $tempSqlFile)" -y | Out-Null

# または、PowerShellのExpand-Archiveを使用（.gzファイルの場合は別の方法が必要）
# gunzipコマンドが利用可能な場合（Git Bashなど）
# gunzip -c $backupFile.FullName > $tempSqlFile

# UTF-8エンコーディングを明示的に設定してリストア
Write-Host "データベースをリストア中（UTF-8エンコーディング設定）..." -ForegroundColor Yellow

# 方法1: PowerShellのGet-Contentを使用（UTF-8を明示的に指定）
$content = Get-Content $tempSqlFile -Encoding UTF8 -Raw
$content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox

# 方法2: ファイルを直接読み込む（Windowsの場合）
# docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox < $tempSqlFile

# 一時ファイルを削除
Remove-Item $tempSqlFile -ErrorAction SilentlyContinue
```

**注意**: 7-Zipがインストールされていない場合、またはgunzipコマンドが利用できない場合は、以下のいずれかの方法を使用してください：

1. **WSL（Windows Subsystem for Linux）を使用**:
   ```powershell
   wsl gunzip -c $backupFile.FullName | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
   ```

2. **Git Bashを使用**:
   ```bash
   # Git Bashで実行
   gunzip -c /c/backup/toybox/database/toybox_*.sql.gz | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
   ```

#### 2-4. リストアの確認

```powershell
# データの件数を確認
docker compose exec db psql -U postgres -d toybox -c "SELECT 'users' as table_name, COUNT(*) as count FROM users UNION ALL SELECT 'submissions', COUNT(*) FROM submissions UNION ALL SELECT 'cards', COUNT(*) FROM gamification_card UNION ALL SELECT 'titles', COUNT(*) FROM gamification_title;"

# 日本語データが正しく表示されるか確認
docker compose exec db psql -U postgres -d toybox -c "SELECT username, display_name FROM users LIMIT 5;"
```

### ステップ3: メディアボリューム（画像ファイル）のリストア

#### 3-1. 現在のメディアボリュームをバックアップ（念のため）

```powershell
# メディアボリュームの内容を確認
docker compose exec web ls -la /app/public/uploads/

# バックアップを作成（オプション）
docker run --rm `
  -v toybox_media_volume:/data `
  -v ${PWD}:/backup `
  alpine tar czf /backup/media_volume_backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').tar.gz /data
```

#### 3-2. メディアボリュームをリストア

```powershell
# 最新のメディアボリュームバックアップを取得
$mediaBackup = Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1

if (-not $mediaBackup) {
    Write-Host "警告: メディアボリュームのバックアップが見つかりません" -ForegroundColor Yellow
    Write-Host "画像ファイルは復元されませんが、データベースのリストアは続行します" -ForegroundColor Yellow
} else {
    Write-Host "メディアボリュームをリストア中: $($mediaBackup.FullName)" -ForegroundColor Yellow
    
    # メディアボリュームをリストア
    docker run --rm `
      -v toybox_media_volume:/data `
      -v "$(Split-Path $mediaBackup.FullName):/backup" `
      alpine sh -c "cd / && tar xzf /backup/$(Split-Path $mediaBackup.FullName -Leaf)"
    
    Write-Host "メディアボリュームのリストアが完了しました" -ForegroundColor Green
}
```

**注意**: Dockerボリューム名が異なる場合は、以下のコマンドで確認してください：

```powershell
# Dockerボリューム一覧を確認
docker volume ls | Select-String "media"
```

ボリューム名が `backend_media_volume` の場合は、上記のコマンドを以下のように変更してください：

```powershell
docker run --rm `
  -v backend_media_volume:/data `
  -v "$(Split-Path $mediaBackup.FullName):/backup" `
  alpine sh -c "cd / && tar xzf /backup/$(Split-Path $mediaBackup.FullName -Leaf)"
```

## トラブルシューティング

### 問題1: 日本語が「???????」と表示される

**原因**: リストア時にUTF-8エンコーディングが設定されていない

**解決方法**:

1. データベースの文字セットを確認：
   ```powershell
   docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
   ```

2. データベースを再作成してUTF-8を明示的に指定：
   ```powershell
   docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
   docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
   ```

3. UTF-8エンコーディングを設定して再リストア：
   ```powershell
   $content = Get-Content $tempSqlFile -Encoding UTF8 -Raw
   $content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
   ```

### 問題2: 画像が表示されない

**原因**: メディアボリュームがリストアされていない

**解決方法**:

1. メディアボリュームのバックアップが存在するか確認：
   ```powershell
   Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz
   ```

2. メディアボリュームをリストア（上記のステップ3-2を参照）

3. 画像ファイルが存在するか確認：
   ```powershell
   docker compose exec web ls -la /app/public/uploads/cards/
   docker compose exec web ls -la /app/public/uploads/titles/
   ```

4. ボリュームが正しくマウントされているか確認：
   ```powershell
   docker compose exec web mount | Select-String "uploads"
   ```

### 問題3: リストア中にエラーが発生する

**原因**: データベースの接続エラーや権限エラー

**解決方法**:

1. Dockerコンテナが起動しているか確認：
   ```powershell
   docker compose ps
   ```

2. PostgreSQLが接続可能か確認：
   ```powershell
   docker compose exec db pg_isready -U postgres
   ```

3. エラーログを確認：
   ```powershell
   docker compose logs db | Select-Object -Last 50
   ```

### 問題4: 7-Zipやgunzipコマンドが利用できない

**解決方法**:

1. **7-Zipをインストール**: https://www.7-zip.org/
2. **WSL（Windows Subsystem for Linux）をインストール**: WSL経由でgunzipコマンドを使用
3. **Git Bashを使用**: Git for Windowsに含まれるGit Bashでgunzipコマンドを使用

## まとめ

この手順に従うことで、以下の問題が解決されます：

✅ **文字化けの問題**: UTF-8エンコーディングを明示的に設定してリストア  
✅ **画像ファイルの問題**: メディアボリュームのバックアップをリストア

**重要なポイント**:

1. データベース作成時に `ENCODING='UTF8'` を明示的に指定
2. リストア時に `PGCLIENTENCODING=UTF8` と `LANG=C.UTF-8` を設定
3. メディアボリュームのバックアップもダウンロードしてリストア
