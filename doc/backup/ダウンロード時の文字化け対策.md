# ダウンロード時の文字化け対策

**作成日**: 2026年1月29日  
**問題**: サーバー上では日本語が正しく表示されるが、ローカルにダウンロードすると文字化けする

---

## 原因

ダウンロード時に文字エンコーディングが変換されている可能性があります。

---

## 解決方法

### 方法1: WinSCPでバイナリモードでダウンロード（推奨）

1. WinSCPを起動
2. サーバーに接続
   - ホスト: `160.251.168.144`
   - ユーザー名: `root`
   - 秘密鍵: `C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem`
3. **オプション** → **環境設定** → **転送** → **テキスト**
   - 「テキストファイルの転送モード」を **「無効」** に設定
4. サーバー側: `/backup/toybox/database/` に移動
5. 最新の `toybox_*.sql.gz` ファイルを右クリック → **ダウンロード**
6. ローカルの `C:\backup\toybox\database\` に保存

**重要**: バイナリモードでダウンロードすることで、ファイルの内容が変換されません。

---

### 方法2: PowerShellでSCPコマンドを修正

PowerShellでは、パスにスペースが含まれる場合やWindowsパスを使用する場合、特別な扱いが必要です。

```powershell
# 方法A: パスを変数に格納
$sshKey = "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem"
$destDir = "C:\backup\toybox\database\"

scp -i $sshKey root@160.251.168.144:/backup/toybox/database/toybox_*.sql.gz $destDir
```

```powershell
# 方法B: WSL経由でダウンロード（Linuxコマンドを使用）
wsl scp -i /mnt/c/Users/ayato/.ssh/toybox-2025-11-06-11-40.pem root@160.251.168.144:/backup/toybox/database/toybox_*.sql.gz /mnt/c/backup/toybox/database/
```

---

### 方法3: サーバー側で直接PostgreSQLから新しいバックアップを取得（最も確実）

この方法では、ダウンロードを経由せずに、サーバー上で直接新しいバックアップを取得してからリストアします。

#### 3-1. サーバーに接続

```bash
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144
```

#### 3-2. サーバー上で新しいバックアップを取得（UTF-8エンコーディング明示）

```bash
# 新しいバックアップを取得
DATE=$(date +%Y%m%d_%H%M%S)
docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 backend-db-1 pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --column-inserts \
    toybox > /tmp/toybox_${DATE}_utf8.sql

# ファイルサイズを確認
ls -lh /tmp/toybox_${DATE}_utf8.sql

# 内容を確認（日本語が正しく表示されるか）
grep "INSERT INTO.*titles" /tmp/toybox_${DATE}_utf8.sql | head -5
```

#### 3-3. 新しいバックアップをダウンロード

```powershell
# PowerShellで実行（ローカル環境）
# 日付とファイル名を確認してから実行
$sshKey = "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem"
$destDir = "C:\backup\toybox\database\"

scp -i $sshKey root@160.251.168.144:/tmp/toybox_*_utf8.sql $destDir
```

#### 3-4. ダウンロードしたファイルの内容を確認

```powershell
# 最新のファイルを取得
$sqlFile = Get-ChildItem C:\backup\toybox\database\toybox_*_utf8.sql | Sort-Object LastWriteTime -Descending | Select-Object -First 1

# 内容を確認（日本語が正しく表示されるか）
Get-Content $sqlFile.FullName -Encoding UTF8 | Select-String "INSERT INTO.*titles" | Select-Object -First 5
```

**期待される結果**: 日本語が正しく表示される

#### 3-5. データベースをリセットしてリストア

```powershell
cd C:\github\toybox\backend

# データベースをリセット
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"

# リストア
$content = Get-Content $sqlFile.FullName -Encoding UTF8 -Raw
$content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```

#### 3-6. リストア結果を確認

```powershell
# 日本語が正しく表示されるか確認
docker compose exec db psql -U postgres -d toybox -c "SELECT id, name FROM gamification_title LIMIT 5;"
```

### 方法4: Git Bashを使用

Git for Windowsに含まれるGit Bashを使用すると、Linuxのコマンドが使えます。

```bash
# Git Bashで実行
scp -i /c/Users/ayato/.ssh/toybox-2025-11-06-11-40.pem root@160.251.168.144:/backup/toybox/database/toybox_*.sql.gz /c/backup/toybox/database/
```

---

## まとめ

ダウンロード時の文字化けを防ぐには：

1. **WinSCPでバイナリモードでダウンロード**（最も簡単）
2. **サーバー上で新しいバックアップを取得**（最も確実）
3. **WSLまたはGit Bashを使用**（Linuxコマンドを使用）

推奨: **方法3（サーバー上で新しいバックアップを取得）**が最も確実です。
