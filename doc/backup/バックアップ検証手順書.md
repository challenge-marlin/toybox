## 概要

この手順書では、**本番サーバーの最新バックアップデータ2つをローカル環境にリストアし、すべてのデータが正しくバックアップされているか検証**します。

### 検証の流れ

```
1. サーバーの最新バックアップを確認
   ↓
2. バックアップファイルをローカルにダウンロード
   ↓
3. ローカル環境にリストア（PostgreSQLダンプ + ボリュームバックアップ）
   ↓
4. すべてのテーブルのデータを検証
   ↓
5. データの整合性を確認
   ↓
6. 検証結果を記録
```

---

## 検証の目的

バックアップは「取得できる」だけでは不十分です。以下の点を確認します：

- ✅ **すべてのテーブルにデータが存在する**
- ✅ **データの件数が期待通りである**
- ✅ **データの整合性が保たれている**（外部キー、制約など）
- ✅ **重要なデータが欠損していない**
- ✅ **復元後にアプリケーションが正常に動作する**

### ステップ1: サーバーの最新バックアップを確認

#### 1-1. サーバーに接続

```powershell
# SSHでサーバーに接続
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144
```

#### 1-2. 最新のバックアップファイルを確認

サーバー側で実行：

```bash
# PostgreSQLダンプの最新を確認
ls -lht /backup/toybox/database/toybox_*.sql.gz | head -1

# ボリューム完全バックアップの最新を確認

ls -lht /backup/toybox/volumes/media_volume_*.tar.gz | head -1

# ボリューム差分バックアップの最新を確認（オプション）
```

**記録**: 最新のバックアップファイル名を控えておきます。

- PostgreSQLダンプ: `toybox_20260127_030000.sql.gz`

### ステップ4: PostgreSQLダンプからリストア

#### 4-1. バックアップファイルを解凍してリストア

```powershell
# バックアップファイルのパスを指定（実際のファイル名に置き換えてください）
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260127_030000.sql.gz"

# ファイルの存在確認
if (-not (Test-Path $BACKUP_FILE)) {
    Write-Host "エラー: バックアップファイルが見つかりません" -ForegroundColor Red
    exit 1
}

# 解凍してリストア
Write-Host "PostgreSQLダンプからリストアを開始します..." -ForegroundColor Yellow

# 7-Zipで解凍
if (Test-Path "C:\Program Files\7-Zip\7z.exe") {
    $tempFile = $BACKUP_FILE -replace "\.gz$", ""
    & "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null
    
    # リストア（エラーを記録するため、ログファイルに出力）
    # 重要: UTF-8を明示的に指定して読み込む（PowerShellの文字エンコーディング問題を回避）
    Write-Host "リストア中（UTF-8で読み込み）..." -ForegroundColor Yellow
    
    $logFile = "$tempFile.log"
    
    # UTF-8を明示的に指定して読み込む
    try {
        $content = [System.IO.File]::ReadAllText($tempFile, [System.Text.Encoding]::UTF8)
        $content | docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile
    } catch {
        Write-Host "エラー: $_" -ForegroundColor Red
        Write-Host "フォールバック: Get-Content -Encoding UTF8を使用します" -ForegroundColor Yellow
        Get-Content $tempFile -Encoding UTF8 | docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile
    }
    
    # エラーログを確認
    $errors = Get-Content $logFile | Select-String -Pattern "ERROR|COPY 0" | Measure-Object
    if ($errors.Count -gt 0) {
        Write-Host "警告: リストア中にエラーが発生しました。ログファイルを確認してください: $logFile" -ForegroundColor Yellow
        Write-Host "エラー数: $($errors.Count)" -ForegroundColor Yellow
        
        # COPY文の成功数を確認
        $copySuccess = Get-Content $logFile | Select-String -Pattern "COPY \d+" | Where-Object { $_ -notmatch "COPY 0" } | Measure-Object
        Write-Host "成功したCOPY文: $($copySuccess.Count)件" -ForegroundColor Cyan
        
        # COPY 0の数を確認
        $copyZero = Get-Content $logFile | Select-String -Pattern "COPY 0" | Measure-Object
        if ($copyZero.Count -gt 0) {
            Write-Host "警告: $($copyZero.Count)件のテーブルでデータがコピーされませんでした（COPY 0）" -ForegroundColor Red
        }
    } else {
        Write-Host "リストアが正常に完了しました" -ForegroundColor Green
    }
    
    # 一時ファイルを削除（ログファイルは残す）
    # Remove-Item $tempFile -ErrorAction SilentlyContinue
    
    Write-Host "PostgreSQLダンプのリストアが完了しました" -ForegroundColor Green
    Write-Host "ログファイル: $logFile" -ForegroundColor Gray
    Write-Host "注意: 'already exists'エラーは正常です。ただし、'COPY 0'やデータ型エラーは問題です。" -ForegroundColor Yellow
} else {
    Write-Host "エラー: 7-Zipが見つかりません" -ForegroundColor Red
    exit 1
}
```

### ステップ5: ボリュームバックアップからリストア（オプション）

**注意**: PostgreSQLダンプでリストアした場合、ボリュームバックアップは通常不要です。  
ただし、メディアファイル（画像・動画）も検証する場合は、ボリュームバックアップからメディアボリュームをリストアしてください。

#### 5-1. メディアボリュームをリストア

```powershell
# バックアップファイルのパスを指定
$MEDIA_BACKUP = "C:\backup\toybox-verification\volumes\media_volume_20260127.tar.gz"

# ファイルの存在確認
if (-not (Test-Path $MEDIA_BACKUP)) {
    Write-Host "警告: メディアボリュームのバックアップファイルが見つかりません" -ForegroundColor Yellow
    Write-Host "スキップします" -ForegroundColor Yellow
} else {
    Write-Host "メディアボリュームをリストアしています..." -ForegroundColor Yellow
    
    # メディアボリュームを削除（存在する場合）
    docker volume rm backend_media_volume -ErrorAction SilentlyContinue
    
    # メディアボリュームを作成
    docker volume create backend_media_volume
    
    # ボリュームをリストア
    docker run --rm `
      -v backend_media_volume:/data `
      -v "$(Split-Path $MEDIA_BACKUP):/backup:ro" `
      alpine sh -c "cd / && tar xzf /backup/$(Split-Path $MEDIA_BACKUP -Leaf)"
    
    Write-Host "メディアボリュームのリストアが完了しました" -ForegroundColor Green
}
```

### ステップ6: リストア結果の確認

リストアが完了したら、エラーが発生していないか確認してください。

#### 6-0. エラーログの確認

```powershell
# ログファイルのパス（ステップ4で作成されたログファイル）
$logFile = "C:\backup\toybox-verification\database\toybox_YYYYMMDD_HHMMSS.log"

# エラーを確認（UTF-8で読み込む）
Write-Host "`n=== エラーログの確認 ===" -ForegroundColor Cyan

$errors = Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" } | Measure-Object
Write-Host "エラー数（already exists以外）: $($errors.Count)件" -ForegroundColor $(if ($errors.Count -eq 0) { "Green" } else { "Yellow" })

if ($errors.Count -gt 0) {
    Write-Host "`nエラー内容（最初の10件）:" -ForegroundColor Yellow
    Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" } | Select-Object -First 10
}

# INSERT文の成功数を確認
$insertSuccess = Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "INSERT \d+ \d+" | Measure-Object
Write-Host "`n成功したINSERT文: $($insertSuccess.Count)件" -ForegroundColor Cyan
```

#### 6-1. 各テーブルのデータ件数を確認

```powershell
# 各テーブルのデータ件数を確認
Write-Host "`n=== データ件数の確認 ===" -ForegroundColor Cyan

docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
UNION ALL
SELECT 'reactions', COUNT(*) FROM reactions
UNION ALL
SELECT 'announcements', COUNT(*) FROM announcements
UNION ALL
SELECT 'titles', COUNT(*) FROM titles
```

**エラーが発生した場合**:
- `COPY 0`となったテーブルがある
- データ型エラーが発生している
- 外部キー制約違反が発生している

→ [リストアエラー診断と対処](./リストアエラー診断と対処.md)を参照してください。

#### 6-2. ユニークインデックスエラーの確認

`titles`テーブルでユニークインデックスエラーが発生した場合、重複データが存在する可能性があります。

```powershell
# titlesテーブルの重複データを確認
Write-Host "`n=== titlesテーブルの重複データ確認 ===" -ForegroundColor Cyan

docker compose exec db psql -U postgres -d toybox -c @"
SELECT name, COUNT(*) as count 
FROM titles 
GROUP BY name 
HAVING COUNT(*) > 1
ORDER BY count DESC;
"@

# すべてのtitlesデータを確認
docker compose exec db psql -U postgres -d toybox -c @"
SELECT id, name, LENGTH(name) as name_length 
FROM titles 
ORDER BY id;
"@
```

**確認事項**:
- ⚠️ **重複データがある場合**: サーバー側のデータベースに既に重複データが存在している可能性があります
- ⚠️ **文字化け（`?`）が表示される場合**: PowerShellのコンソールエンコーディングの問題の可能性があります。データベース内のデータ自体は正しい可能性があります

**対処方法**:
1. サーバー側のデータベースで実際のデータを確認（下記参照）
2. 重複データがサーバー側に存在する場合は、データクリーンアップが必要です
3. 文字化けはコンソール表示の問題で、データ自体は正しい可能性が高いです

#### 6-3. サーバー側のデータベースで実際のデータを確認

ローカルで重複データが検出された場合、サーバー側のデータベースで実際のデータを確認してください。

```bash
# サーバーにSSH接続後、以下を実行
docker exec backend-db-1 psql -U postgres -d toybox -c @"
SELECT id, name, LENGTH(name) as name_length 
FROM titles 
ORDER BY id;
"@

# 重複データを確認
docker exec backend-db-1 psql -U postgres -d toybox -c @"
SELECT name, COUNT(*) as count 
FROM titles 
GROUP BY name 
HAVING COUNT(*) > 1
ORDER BY count DESC;
"@
```

**確認事項**:
- ✅ サーバー側でも重複データが存在する場合 → データクリーンアップが必要
- ✅ サーバー側では重複がない場合 → バックアップ/リストア時の文字エンコーディング問題の可能性

**原因**: バックアップファイルにCREATE TABLE文が含まれており、既にテーブルが存在する場合に発生します。

**解決方法**:

- データが正しくリストアされているか、ステップ6のデータ検証で確認してください。

### 問題2: データ件数が0件、または期待より少ない（重要！）

**症状**: 
- `users`テーブルにはデータがあるが、他のテーブル（`cards`, `submissions`, `user_cards`など）が0件
- 例: `users: 12件`、`cards: 0件`、`submissions: 0件`

**原因**: 
- リストア時にエラーが発生し、データのCOPY部分が失敗している
- バックアップファイルのデータ型と現在のスキーマが一致していない
- 外部キー制約違反により、データのコピーが途中で停止している

**診断方法**:
1. **リストアログファイルを確認**:
   ```powershell

   # エラーを確認
   Get-Content $logFile | Select-String -Pattern "ERROR"
   
   # COPY 0の数を確認（データがコピーされなかったテーブル）
   Get-Content $logFile | Select-String -Pattern "COPY 0"
   
   # 成功したCOPY文を確認
   Get-Content $logFile | Select-String -Pattern "COPY \d+" | Where-Object { $_ -notmatch "COPY 0" }
   ```

2. **バックアップファイルの内容を確認**:
   ```powershell
   # バックアップファイルを解凍
   $BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260127_030000.sql.gz"

   & "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y

   Get-Content $tempFile | Select-String -Pattern "^COPY " | ForEach-Object {
   ```
