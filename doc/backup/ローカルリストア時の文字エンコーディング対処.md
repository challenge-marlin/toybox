# ローカルリストア時の文字エンコーディング対処

**作成日**: 2026年1月28日  
**目的**: ローカル環境でリストアする際の文字エンコーディング問題を解決する

---

## 問題の原因

サーバー側のバックアップファイルには正しい日本語が含まれていますが、ローカル環境でリストアする際に文字化けが発生しています。

**原因**: PowerShellの`Get-Content`コマンドが、デフォルトでシステムのコードページ（日本語Windowsでは通常Shift-JIS）を使用しているため、UTF-8ファイルを正しく読み込めていない可能性があります。

---

## 解決方法

### 方法1: Get-ContentでUTF-8を明示的に指定（推奨）

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# データベースをリセット
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"

# バックアップファイルからリストア（UTF-8を明示的に指定）
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260128_150118.sql.gz"
$tempFile = $BACKUP_FILE -replace "\.gz$", ""
$logFile = "$tempFile.log"

Write-Host "バックアップファイルを解凍中..." -ForegroundColor Yellow
& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null

Write-Host "データベースにリストア中（UTF-8で読み込み）..." -ForegroundColor Yellow

# UTF-8を明示的に指定して読み込む
[System.IO.File]::ReadAllText($tempFile, [System.Text.Encoding]::UTF8) | 
    docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile

Write-Host "`nリストアが完了しました" -ForegroundColor Green
```

### 方法2: chcpコマンドでコードページを変更

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# コードページをUTF-8に変更（65001はUTF-8のコードページ）
chcp 65001

# データベースをリセット
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"

# バックアップファイルからリストア
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260128_150118.sql.gz"
$tempFile = $BACKUP_FILE -replace "\.gz$", ""
$logFile = "$tempFile.log"

& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null

Get-Content $tempFile -Encoding UTF8 | docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile
```

### 方法3: Dockerコンテナ内で直接リストア

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# データベースをリセット
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"

# バックアップファイルをDockerコンテナにコピー
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260128_150118.sql.gz"
$tempFile = $BACKUP_FILE -replace "\.gz$", ""

& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null

# Dockerコンテナ内の一時ファイルにコピー
docker cp $tempFile backend-db-1:/tmp/restore.sql

# Dockerコンテナ内で直接リストア（エンコーディングの問題を回避）
docker compose exec db psql -U postgres -d toybox -f /tmp/restore.sql

# 一時ファイルを削除
docker compose exec db rm /tmp/restore.sql
```

## 推奨される方法

**方法1（`[System.IO.File]::ReadAllText`を使用）**を推奨します。理由：
- PowerShellのエンコーディング問題を完全に回避できる
- UTF-8を明示的に指定できる
- 最も確実

---

## 完全なリストアスクリプト（推奨版）

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# データベースをリセット
Write-Host "データベースをリセット中..." -ForegroundColor Yellow
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"

# バックアップファイルからリストア
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260128_150118.sql.gz"
$tempFile = $BACKUP_FILE -replace "\.gz$", ""
$logFile = "$tempFile.log"

Write-Host "`nバックアップファイルを解凍中..." -ForegroundColor Yellow
& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null

Write-Host "データベースにリストア中（UTF-8で読み込み）..." -ForegroundColor Yellow

# UTF-8を明示的に指定して読み込む
try {
    $content = [System.IO.File]::ReadAllText($tempFile, [System.Text.Encoding]::UTF8)
    $content | docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile
    
    Write-Host "`nリストアが完了しました" -ForegroundColor Green
} catch {
    Write-Host "エラー: $_" -ForegroundColor Red
    exit 1
}

# リストア結果を確認
Write-Host "`n=== リストア結果の確認 ===" -ForegroundColor Cyan

# エラーを確認
$errors = Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" } | Measure-Object
Write-Host "エラー数（already exists以外）: $($errors.Count)件" -ForegroundColor $(if ($errors.Count -eq 0) { "Green" } else { "Yellow" })

# データ件数を確認
Write-Host "`n=== データ件数の確認 ===" -ForegroundColor Cyan
docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
UNION ALL
SELECT 'announcements', COUNT(*) FROM announcements
UNION ALL
SELECT 'titles', COUNT(*) FROM titles
UNION ALL
SELECT 'user_meta', COUNT(*) FROM user_meta
ORDER BY table_name;
"@
```
