# バックアップファイル検証コマンド集

**作成日**: 2026年1月28日  
**目的**: サーバー側でバックアップファイルの内容を詳細に確認するコマンド集

---

## 基本的な確認コマンド

### 1. 最新のバックアップファイルを確認

```bash
# 最新のバックアップファイルを取得
LATEST_BACKUP=$(ls -t /backup/toybox/database/toybox_*.sql.gz | head -1)
echo "Checking: $LATEST_BACKUP"

# ファイルサイズを確認
ls -lh "$LATEST_BACKUP"
```

### 2. INSERT文の総数を確認

```bash
gunzip -c "$LATEST_BACKUP" | grep -c "^INSERT INTO "
```

### 3. 主要テーブルごとのINSERT文の数を確認

```bash
echo "=== 主要テーブルのINSERT文の数 ==="
for TABLE in users cards submissions user_cards announcements titles user_meta reactions; do
    COUNT=$(gunzip -c "$LATEST_BACKUP" | grep -c "INSERT INTO.*${TABLE}" 2>/dev/null || echo "0")
    echo "${TABLE}: ${COUNT}件"
done
```

### 4. 各テーブルのINSERT文のサンプルを確認

```bash
echo "=== usersテーブルのINSERT文サンプル ==="
gunzip -c "$LATEST_BACKUP" | grep "INSERT INTO.*users" | head -3

echo "=== cardsテーブルのINSERT文サンプル ==="
gunzip -c "$LATEST_BACKUP" | grep "INSERT INTO.*cards" | head -3

echo "=== submissionsテーブルのINSERT文サンプル ==="
gunzip -c "$LATEST_BACKUP" | grep "INSERT INTO.*submissions" | head -3
```

### 5. バックアップファイルの文字エンコーディングを確認

```bash
# ファイルの先頭部分を確認（文字化けがないか）
gunzip -c "$LATEST_BACKUP" | head -50 | file -
```

### 6. バックアップファイルの構造を確認

```bash
# CREATE TABLE文の数を確認
gunzip -c "$LATEST_BACKUP" | grep -c "^CREATE TABLE"

# ALTER TABLE文の数を確認
gunzip -c "$LATEST_BACKUP" | grep -c "^ALTER TABLE"

# セクションの確認
gunzip -c "$LATEST_BACKUP" | grep -E "^(SET|CREATE|ALTER|INSERT|COPY)" | head -20
```

## 詳細な検証コマンド

### すべてのテーブルのINSERT文を一覧表示

```bash
echo "=== すべてのテーブルのINSERT文の数 ==="
gunzip -c "$LATEST_BACKUP" | grep "^INSERT INTO " | sed 's/INSERT INTO \([^(]*\).*/\1/' | sort | uniq -c | sort -rn
```

### データ行数の推定

```bash
# INSERT文の総数（データ行数と一致）
echo "総データ行数（推定）: $(gunzip -c "$LATEST_BACKUP" | grep -c "^INSERT INTO ")行"
```

## 期待される結果

### 正常なバックアップの場合

- **INSERT文の総数**: 100以上（データ量に応じて）
- **主要テーブルのINSERT文**:
  - `users`: 10件以上
  - `cards`: 50件以上
  - `submissions`: 20件以上
  - `user_cards`: 30件以上
  - `announcements`: 1件以上
  - `titles`: 10件以上
  - `user_meta`: 10件以上

### 問題がある場合

- INSERT文の総数が10未満
- 主要テーブルのINSERT文が0件
- 文字化けが発生している
