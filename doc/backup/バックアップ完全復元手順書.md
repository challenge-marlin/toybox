## 概要

この手順書では、**本番環境（ConoHaサーバー）で何かあったときに、本番環境でバックアップから復元する手順**を説明します。

### 使用場面

- データベースが破損した場合
- 誤ってデータを削除してしまった場合
- システム障害でデータが失われた場合
- 特定の日時の状態に戻したい場合

### 注意事項

⚠️ **本番環境での復元は慎重に行ってください**  
⚠️ **復元前に必ず現在のデータベースをバックアップしてください**  
⚠️ **復元はデータベース全体を上書きします**

### 重要な原則

> **バックアップは「取得できる」だけでは不十分です。**  
> **「復元できる」ことが確認できて初めて意味があります。**

### 前提条件

- ✅ 本番環境で実際にバックアップを取得済み
- ✅ サーバーにSSHまたはVNCコンソールでアクセス可能
- ✅ サーバー側でDocker環境が動作している
- ✅ 復元前に現在のデータベースをバックアップ済み（重要！）

### 関連ドキュメント

- **バックアップ検証手順書**: サーバーの最新バックアップをローカル環境にリストアして検証する手順
- **バックアップ実装ガイド**: バックアップシステムの実装方法

## バックアップの種類

TOYBOXでは以下の3種類のバックアップを取得しています：

| 種類 | 保存先 | ファイル名形式 | 頻度 | 保持期間 |
|------|--------|----------------|------|----------|
| **PostgreSQLダンプ** | `/backup/toybox/database/` | `toybox_YYYYMMDD_HHMMSS.sql.gz` | 毎日 3:00 | 7日 |
| **ボリューム完全** | `/backup/toybox/volumes/` | `postgres_data_YYYYMMDD.tar.gz`<br>`media_volume_YYYYMMDD.tar.gz` | 毎週日曜 3:30 | 14日 |
| **ボリューム差分** | `/backup/toybox/volumes-incremental/` | `base_YYYYMMDD/`<br>`incr_YYYYMMDD/` | 毎日 3:45 | 14日 |

### 各バックアップ方式の特徴

| 方式 | サイズ | 復元速度 | 確実性 | 推奨用途 |
|------|--------|----------|--------|----------|
| PostgreSQLダンプ | 小 | 速い | 高 | **日常的な復元（推奨）** |
| ボリューム完全 | 大 | 中 | 高 | 週次バックアップ、完全復旧 |
| ボリューム差分 | 小 | やや遅い | 中 | 日次バックアップ、特定日時への復元 |

## サーバーからバックアップファイルを取得

### ステップ1: サーバーに接続してバックアップファイルを確認

#### 方法A: SSH接続（推奨）

```powershell
# SSHでサーバーに接続
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144
```

#### 方法B: VNCコンソール経由

1. ConoHaの管理画面にログイン
2. サーバー一覧から該当サーバー（160.251.168.144）を選択
3. 「VNCコンソール」または「コンソール」をクリック
4. `root`ユーザーでログイン

### ステップ2: バックアップファイルの一覧を確認

サーバー側で以下のコマンドを実行：

```bash
# PostgreSQLダンプの一覧
ls -lh /backup/toybox/database/toybox_*.sql.gz

# ボリューム完全バックアップの一覧
ls -lh /backup/toybox/volumes/postgres_data_*.tar.gz
ls -lh /backup/toybox/volumes/media_volume_*.tar.gz

# ボリューム差分バックアップの一覧
ls -ld /backup/toybox/volumes-incremental/base_*
ls -ld /backup/toybox/volumes-incremental/incr_*
```

**昨日までのバックアップを確認**（例：今日が1月27日の場合、1月26日までのバックアップを確認）：

```bash
# 昨日の日付を指定（例：20260126）
DATE_YESTERDAY="20260126"

# PostgreSQLダンプ（昨日のもの）
ls -lh /backup/toybox/database/toybox_${DATE_YESTERDAY}*.sql.gz

# ボリューム完全バックアップ（昨日のもの）
ls -lh /backup/toybox/volumes/postgres_data_${DATE_YESTERDAY}.tar.gz
ls -lh /backup/toybox/volumes/media_volume_${DATE_YESTERDAY}.tar.gz

# ボリューム差分バックアップ（昨日のもの）
ls -ld /backup/toybox/volumes-incremental/base_${DATE_YESTERDAY}
ls -ld /backup/toybox/volumes-incremental/incr_${DATE_YESTERDAY}
```

### ステップ3: バックアップファイルをローカルにダウンロード

#### 方法1: WinSCPを使用（推奨・簡単）

1. **WinSCPを起動**
2. **接続情報を入力**：
   - ファイルプロトコル: `SFTP`
   - ホスト名: `160.251.168.144`
   - ポート番号: `22`
   - ユーザー名: `root`
   - 秘密鍵: `C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem`
3. **接続後、以下のファイルをダウンロード**：

   **PostgreSQLダンプ**:
   ```
   サーバー側: /backup/toybox/database/toybox_YYYYMMDD_HHMMSS.sql.gz
   ↓
   ローカル側: C:\backup\toybox-restore-test\database\toybox_YYYYMMDD_HHMMSS.sql.gz
   ```

   **ボリューム完全バックアップ**:
   ```
   サーバー側: /backup/toybox/volumes/postgres_data_YYYYMMDD.tar.gz
   サーバー側: /backup/toybox/volumes/media_volume_YYYYMMDD.tar.gz
   ↓
   ローカル側: C:\backup\toybox-restore-test\volumes\postgres_data_YYYYMMDD.tar.gz
   ローカル側: C:\backup\toybox-restore-test\volumes\media_volume_YYYYMMDD.tar.gz
   ```

   **ボリューム差分バックアップ**（ディレクトリごと）:
   ```
   サーバー側: /backup/toybox/volumes-incremental/base_YYYYMMDD/
   サーバー側: /backup/toybox/volumes-incremental/incr_YYYYMMDD/
   ↓
   ローカル側: C:\backup\toybox-restore-test\volumes-incremental\base_YYYYMMDD\
   ローカル側: C:\backup\toybox-restore-test\volumes-incremental\incr_YYYYMMDD\
   ```

#### 方法2: SCPコマンドを使用

PowerShellで実行：

```powershell
# ダウンロード先ディレクトリを作成
New-Item -ItemType Directory -Path "C:\backup\toybox-restore-test\database" -Force
New-Item -ItemType Directory -Path "C:\backup\toybox-restore-test\volumes" -Force
New-Item -ItemType Directory -Path "C:\backup\toybox-restore-test\volumes-incremental" -Force

# PostgreSQLダンプをダウンロード（例：2026年1月26日の最新）
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/database/toybox_20260126*.sql.gz `
  C:\backup\toybox-restore-test\database\

# ボリューム完全バックアップをダウンロード
scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes/postgres_data_20260126.tar.gz `
  C:\backup\toybox-restore-test\volumes\

scp -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes/media_volume_20260126.tar.gz `
  C:\backup\toybox-restore-test\volumes\

# ボリューム差分バックアップをダウンロード（ディレクトリごと）
scp -r -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes-incremental/base_20260126 `
  C:\backup\toybox-restore-test\volumes-incremental\

scp -r -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" `
  root@160.251.168.144:/backup/toybox/volumes-incremental/incr_20260126 `
  C:\backup\toybox-restore-test\volumes-incremental\
```

### ステップ4: ダウンロードしたファイルを確認

```powershell
# ファイルの存在確認
dir C:\backup\toybox-restore-test\database\
dir C:\backup\toybox-restore-test\volumes\
dir C:\backup\toybox-restore-test\volumes-incremental\

# ファイルサイズを確認（0バイトでないことを確認）
```

**確認事項**:
- ✅ ファイルが存在する
- ✅ ファイルサイズが0より大きい
- ✅ ファイル名が正しい（日付が一致している）

## PostgreSQLダンプからの復元

**推奨**: 最も簡単で高速な復元方法です。日常的な復元に最適です。

### ステップ1: 復元前のデータを確認（比較用）

```powershell
# 現在のユーザー数を確認
docker exec backend-db-1 psql -U postgres toybox -t -c "SELECT COUNT(*) FROM users;"

# 現在の投稿数を確認
docker exec backend-db-1 psql -U postgres toybox -t -c "SELECT COUNT(*) FROM submissions;"

# 現在のカード数を確認
docker exec backend-db-1 psql -U postgres toybox -t -c "SELECT COUNT(*) FROM cards;"
```

**メモ**: これらの数値を控えておいてください。復元後に数値が変わることを確認します。

### ステップ2: 現在のデータベースをバックアップ（念のため）

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# 現在のデータベースをバックアップ
$backupName = "backup_before_restore_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql"
docker compose exec -T db pg_dump -U postgres toybox > $backupName

Write-Host "バックアップ完了: $backupName" -ForegroundColor Green
```

### ステップ3: データベースをリセット（オプション）

**注意**: 既存のデータを完全に上書きする場合は、データベースを削除して再作成します。

```powershell
# 警告: これにより現在のデータベースが削除されます
$confirm = Read-Host "データベースを削除して再作成しますか？ (yes/no)"
if ($confirm -eq "yes") {
    # データベースを削除
    docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
    
    # データベースを再作成
    docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"
    
    Write-Host "データベースをリセットしました" -ForegroundColor Green
}
```

### ステップ4: バックアップから復元

#### 方法1: 圧縮ファイルから直接復元（推奨）

```powershell
# バックアップファイルのパスを指定（実際のファイル名に置き換えてください）
$BACKUP_FILE = "C:\backup\toybox-restore-test\database\toybox_20260126_170000.sql.gz"

# ファイルの存在確認
if (-not (Test-Path $BACKUP_FILE)) {
    Write-Host "エラー: バックアップファイルが見つかりません" -ForegroundColor Red
    exit 1
}

# 解凍しながら復元
Write-Host "復元を開始します..." -ForegroundColor Yellow

# 7-Zipで解凍してから復元
if (Test-Path "C:\Program Files\7-Zip\7z.exe") {
    $tempFile = $BACKUP_FILE -replace "\.gz$", ""
    & "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null
    
    Get-Content $tempFile | docker compose exec -T db psql -U postgres toybox
    
    # 一時ファイルを削除
    Remove-Item $tempFile -ErrorAction SilentlyContinue
} else {
    Write-Host "エラー: 7-Zipが見つかりません" -ForegroundColor Red
    Write-Host "手動で解凍してから復元してください" -ForegroundColor Yellow
    exit 1
}

Write-Host "復元が完了しました" -ForegroundColor Green
```

#### 方法2: 一度解凍してから復元

```powershell
# 1. 解凍（7-Zipが必要）
$BACKUP_FILE = "C:\backup\toybox-restore-test\database\toybox_20260126_170000.sql.gz"
$EXTRACTED_FILE = $BACKUP_FILE -replace "\.gz$", ""

& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $EXTRACTED_FILE)" -y

# 2. 復元
Get-Content $EXTRACTED_FILE | docker compose exec -T db psql -U postgres toybox

# 3. 解凍ファイルを削除（オプション）
# Remove-Item $EXTRACTED_FILE
```

## Dockerボリューム完全バックアップからの復元

**用途**: データベース全体を完全に復旧する場合。PostgreSQLダンプで復元できない場合の最終手段。

### ステップ1: 復元前の準備

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# 現在のデータベースをバックアップ（念のため）
$backupName = "backup_before_restore_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql"
docker compose exec -T db pg_dump -U postgres toybox > $backupName
```

### ステップ2: データベースコンテナを停止

```powershell
# 警告: データベースコンテナを停止します
$confirm = Read-Host "データベースコンテナを停止しますか？ (yes/no)"
if ($confirm -ne "yes") {
    Write-Host "キャンセルしました" -ForegroundColor Yellow
    exit 0
}

# データベースコンテナを停止
docker compose stop db

Write-Host "データベースコンテナを停止しました" -ForegroundColor Green
```

### ステップ3: 既存のボリュームを削除

```powershell
# 警告: 既存のボリュームを削除します
$confirm = Read-Host "既存のボリュームを削除しますか？ (yes/no)"
if ($confirm -ne "yes") {
    Write-Host "キャンセルしました" -ForegroundColor Yellow
    docker compose start db
    exit 0
}

# ボリュームを削除
docker volume rm backend_postgres_data

Write-Host "ボリュームを削除しました" -ForegroundColor Green
```

### ステップ4: バックアップからボリュームを復元

```powershell
# バックアップファイルのパスを指定（実際のファイル名に置き換えてください）
$BACKUP_FILE = "C:\backup\toybox-restore-test\volumes\postgres_data_20260126.tar.gz"

# ファイルの存在確認
if (-not (Test-Path $BACKUP_FILE)) {
    Write-Host "エラー: バックアップファイルが見つかりません" -ForegroundColor Red
    exit 1
}

# ボリュームを復元
Write-Host "ボリュームを復元しています..." -ForegroundColor Yellow

# Dockerコンテナを使用して復元
docker run --rm `
  -v backend_postgres_data:/data `
  -v "$(Split-Path $BACKUP_FILE):/backup:ro" `
  alpine sh -c "cd / && tar xzf /backup/$(Split-Path $BACKUP_FILE -Leaf)"

Write-Host "ボリュームの復元が完了しました" -ForegroundColor Green
```

### ステップ5: データベースコンテナを起動

```powershell
# データベースコンテナを起動
docker compose start db

Write-Host "データベースコンテナを起動しました" -ForegroundColor Green
Write-Host "データベースが起動するまで約30秒お待ちください..." -ForegroundColor Yellow

Start-Sleep -Seconds 30
```

### ステップ6: 復元結果を確認

[復元検証手順](#復元検証手順)を参照してください。

## Dockerボリューム差分バックアップからの復元

**用途**: 特定の日時（例：昨日）の状態に復元する場合。ベースバックアップと差分バックアップを組み合わせて復元します。

### ステップ1: 復元前の準備

```powershell

cd C:\github\toybox\backend

# 現在のデータベースをバックアップ（念のため）
$backupName = "backup_before_restore_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql"
docker compose exec -T db pg_dump -U postgres toybox > $backupName
```

### ステップ2: 復元する日付を指定

```powershell
# 復元する日付を入力（例：20260126）
$TARGET_DATE = Read-Host "復元する日付を入力してください (例: 20260126)"

# バックアップディレクトリのパス
$BASE_DIR = "C:\backup\toybox-restore-test\volumes-incremental\base_$TARGET_DATE"
$INCR_DIR = "C:\backup\toybox-restore-test\volumes-incremental\incr_$TARGET_DATE"

# バックアップディレクトリの存在確認
if (-not (Test-Path $BASE_DIR)) {
    Write-Host "エラー: ベースバックアップが見つかりません: $BASE_DIR" -ForegroundColor Red
    exit 1
}

if (-not (Test-Path $INCR_DIR)) {
    Write-Host "エラー: 差分バックアップが見つかりません: $INCR_DIR" -ForegroundColor Red
    exit 1
}

Write-Host "ベースバックアップ: $BASE_DIR" -ForegroundColor Green
Write-Host "差分バックアップ: $INCR_DIR" -ForegroundColor Green
```

### ステップ3: データベースコンテナを停止

```powershell
# 警告: データベースコンテナを停止します
```

### ステップ4: 既存のボリュームを削除

## 復元検証手順

復元が成功したことを確認するための手順です。

### ステップ1: データベースの接続確認

```powershell
# データベースに接続できるか確認
docker compose exec db psql -U postgres -d toybox -c "SELECT version();"
```

**期待される結果**: PostgreSQLのバージョン情報が表示される

### ステップ2: データの件数を確認

```powershell
# 各テーブルの件数を確認
docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'reactions', COUNT(*) FROM reactions
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
```

## トラブルシューティング

### 問題1: 「gunzipコマンドが見つかりません」エラー

**原因**: Dockerコンテナ内にgunzipコマンドがない

**解決方法**: 7-Zipを使用してローカルで解凍してから復元してください（[PostgreSQLダンプからの復元 - 方法2](#方法2-一度解凍してから復元)を参照）

---

### 問題2: 「relation "users" already exists」エラーが大量に出る

**原因**: 既存のテーブルを上書きしようとしているため

**解決方法**: これは正常です。既存のテーブルを上書きしているためです。最後に「COPY XXXX」と表示されれば成功しています。

完全にクリーンな状態から復元したい場合は、[PostgreSQLダンプからの復元 - ステップ3](#ステップ3-データベースをリセットオプション)でデータベースをリセットしてから復元してください。

---

### 問題3: 復元後にデータが変わっていない

**原因**: 復元コマンドが失敗している可能性があります

**解決方法**:
1. バックアップファイルが正しくダウンロードされているか確認
2. ファイルサイズを確認（0バイトでないか）
3. 復元コマンドのエラーメッセージを確認
4. 復元コマンドを再実行

---

### 問題4: 「FATAL: database "toybox" does not exist」エラー

**原因**: データベースが存在しない

**解決方法**: データベースを作成してください

```powershell
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"
```

---

### 問題5: ボリューム復元時に「volume backend_postgres_data not found」エラー

**原因**: ボリュームが存在しない

**解決方法**: ボリュームを先に作成してください

```powershell
# ボリュームを作成
docker volume create backend_postgres_data

# その後、復元を実行
```

---

### 問題6: 差分バックアップの復元時に「base_YYYYMMDDが見つかりません」エラー

**原因**: ベースバックアップが存在しない、または日付が間違っている

**解決方法**:
1. ベースバックアップのディレクトリが存在するか確認
2. 日付が正しいか確認（YYYYMMDD形式）
3. ベースバックアップが復元対象の日付より古いことを確認

---

### 問題7: 復元後にWebアプリケーションが起動しない

**原因**: データベースのマイグレーションが適用されていない可能性

**解決方法**: マイグレーションを実行してください

```powershell
cd C:\github\toybox\backend
docker compose exec web python3 manage.py migrate
```

---

### 問題8: ファイルサイズが0バイト

**原因**: バックアップファイルのダウンロードが失敗している

**解決方法**:
1. サーバー側でバックアップファイルが正しく存在するか確認
2. ファイル転送を再実行
3. ディスク容量を確認
