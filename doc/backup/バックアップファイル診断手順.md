# バックアップファイル診断手順

**作成日**: 2026年1月28日  
**目的**: バックアップファイルの内容を確認して、問題の原因を特定する

---

## 現在の状況

修正後のスクリプトで作成したバックアップファイルでも、同じエラーが発生しています：
- 文字エンコーディングの問題（文字化け）
- 列の順序の問題（データ型エラー）
- 外部キー制約違反

これは、**サーバー側のデータベース自体に問題がある可能性**があります。

---

## 診断手順

### ステップ1: サーバー側でバックアップファイルの内容を確認

```bash
# サーバー側で実行
LATEST_BACKUP=$(ls -t /backup/toybox/database/toybox_*.sql.gz | head -1)
echo "Checking: $LATEST_BACKUP"

# バックアップファイルを解凍
gunzip -c "$LATEST_BACKUP" > /tmp/backup_check.sql

# announcementsテーブルのCOPY文を確認（列の順序）
echo "=== announcementsテーブルのCOPY文 ==="
grep -A 5 "^COPY announcements" /tmp/backup_check.sql | head -10

# cardsテーブルのCOPY文を確認
echo "=== cardsテーブルのCOPY文 ==="
grep -A 5 "^COPY cards" /tmp/backup_check.sql | head -10

# titlesテーブルのCOPY文を確認
echo "=== titlesテーブルのCOPY文 ==="
grep -A 5 "^COPY titles" /tmp/backup_check.sql | head -10

# user_metaテーブルのCOPY文を確認
echo "=== user_metaテーブルのCOPY文 ==="
grep -A 5 "^COPY user_meta" /tmp/backup_check.sql | head -10
```

### ステップ2: サーバー側のデータベースのスキーマを確認

```bash
# サーバー側で実行
cd /var/www/toybox/backend

# announcementsテーブルのスキーマを確認
docker compose exec db psql -U postgres -d toybox -c "\d announcements"

# cardsテーブルのスキーマを確認
docker compose exec db psql -U postgres -d toybox -c "\d cards"

# titlesテーブルのスキーマを確認
docker compose exec db psql -U postgres -d toybox -c "\d titles"

# user_metaテーブルのスキーマを確認
docker compose exec db psql -U postgres -d toybox -c "\d user_meta"
```

### ステップ3: サーバー側のデータベースの実際のデータを確認

```bash
# サーバー側で実行
cd /var/www/toybox/backend

# announcementsテーブルのデータを確認
docker compose exec db psql -U postgres -d toybox -c "SELECT id, title, created_by_id FROM announcements LIMIT 1;"

# cardsテーブルのデータを確認
docker compose exec db psql -U postgres -d toybox -c "SELECT code, name, rarity, updated_at FROM cards LIMIT 1;"

# titlesテーブルのデータを確認
docker compose exec db psql -U postgres -d toybox -c "SELECT name, duration_days, created_at FROM titles LIMIT 1;"

# user_metaテーブルのデータを確認
docker compose exec db psql -U postgres -d toybox -c "SELECT user_id, updated_at FROM user_meta LIMIT 1;"
```

## 考えられる原因と対処方法

### 原因1: サーバー側のデータベースに既に問題がある

**症状**: サーバー側のデータベース自体に、列の順序が間違っている、またはデータが破損している

**確認方法**:
```bash
# サーバー側で実行
docker compose exec db psql -U postgres -d toybox -c "SELECT * FROM announcements LIMIT 1;"
```

**対処方法**: サーバー側のデータベースを修復する必要があります。

### 原因2: pg_dumpのオプションが不足している

**症状**: `--column-inserts`オプションがないため、列の順序が正しくない

**対処方法**: バックアップスクリプトに`--column-inserts`オプションを追加：

```bash
# 修正後のpg_dumpコマンド
docker exec backend-db-1 pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --verbose \
    --column-inserts \
    toybox 2>> $LOGFILE | gzip > $BACKUP_DIR/toybox_$DATE.sql.gz
```

**注意**: `--column-inserts`オプションを使用すると、バックアップファイルが大きくなり、リストアが遅くなる可能性があります。

### 原因3: データベースの文字エンコーディングがUTF-8ではない

**確認方法**:
```bash
# サーバー側で実行
docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
```

**期待される結果**: `UTF8`

**対処方法**: データベースの文字エンコーディングがUTF-8でない場合は、データベースを再作成する必要があります。

## 推奨される対処方法

### 方法1: `--column-inserts`オプションを追加してバックアップを再作成（推奨）

列の順序の問題を解決するために、`--column-inserts`オプションを追加します。

**手順**:

1. **バックアップスクリプトを修正**:
   ```bash
   # サーバー側で実行
   vi /var/www/toybox/scripts/backup_database.sh
   ```
   
   以下の行を修正：
   ```bash
   # 修正前
   docker exec backend-db-1 pg_dump \
       -U postgres \
       --encoding=UTF8 \
       --no-owner \
       --no-privileges \
       --clean \
       --if-exists \
       --verbose \
       toybox
   
   # 修正後（--column-insertsを追加）
   docker exec backend-db-1 pg_dump \
       -U postgres \
       --encoding=UTF8 \
       --no-owner \
       --no-privileges \
       --clean \
       --if-exists \
       --verbose \
       --column-inserts \
       toybox
   ```

2. **新しいバックアップを作成**:
   ```bash
   /var/www/toybox/scripts/backup_database.sh
   ```

3. **新しいバックアップファイルをローカルにダウンロードして再リストア**

### 方法2: サーバー側のデータベースを直接確認

サーバー側のデータベースに問題がある可能性があるため、直接確認してください：

```bash
# サーバー側で実行
cd /var/www/toybox/backend

# 各テーブルのデータ件数を確認
docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
UNION ALL
SELECT 'announcements', COUNT(*) FROM announcements
UNION ALL
SELECT 'titles', COUNT(*) FROM titles
UNION ALL
SELECT 'user_meta', COUNT(*) FROM user_meta
ORDER BY table_name;
"@
```
