# リストア確認手順

**作成日**: 2026年1月29日  
**目的**: リストアしたデータベースの内容を確認する

---

## ステップ1: テーブル一覧を確認

```powershell
docker compose exec db psql -U postgres -d toybox -c "\dt"
```

このコマンドで、データベースに存在するテーブルの一覧が表示されます。

---

## ステップ2: テーブルの構造を確認

```powershell
# usersテーブルの構造を確認
docker compose exec db psql -U postgres -d toybox -c "\d users"

# 称号テーブルの構造を確認（テーブル名を確認してから実行）
docker compose exec db psql -U postgres -d toybox -c "\d gamification_title"
```

---

## ステップ3: データを確認（正しいカラム名で実行）

### usersテーブル

```powershell
# usersテーブルの全カラムを確認
docker compose exec db psql -U postgres -d toybox -c "SELECT * FROM users LIMIT 1;"

# カラム名を確認してからデータを表示
docker compose exec db psql -U postgres -d toybox -c "SELECT id, email, name FROM users LIMIT 5;"
```

### 称号テーブル

```powershell
# 称号テーブルのデータを確認（テーブル名を確認してから実行）
docker compose exec db psql -U postgres -d toybox -c "SELECT * FROM gamification_title LIMIT 1;"
```

---

## ステップ4: データ件数を確認

```powershell
# usersテーブルの件数
docker compose exec db psql -U postgres -d toybox -c "SELECT COUNT(*) FROM users;"

# cardsテーブルの件数（存在する場合）
docker compose exec db psql -U postgres -d toybox -c "SELECT COUNT(*) FROM gamification_card;"
```

---

## ステップ5: 日本語データを確認

正しいテーブル名とカラム名を使用して、日本語が正しく表示されるか確認します。

```powershell
# usersテーブルの日本語データを確認（カラム名を置き換える）
docker compose exec db psql -U postgres -d toybox -c "SELECT id, name FROM users LIMIT 5;"
```

---

## トラブルシューティング

### 問題1: テーブルが存在しない

リストアが正しく実行されていない可能性があります。以下を確認してください：

```powershell
# データベースの一覧を確認
docker compose exec db psql -U postgres -c "\l"

# toyboxデータベースのサイズを確認
docker compose exec db psql -U postgres -c "SELECT pg_database.datname, pg_size_pretty(pg_database_size(pg_database.datname)) AS size FROM pg_database WHERE datname = 'toybox';"
```

サイズが非常に小さい（例: 8MB以下）場合、リストアが失敗している可能性があります。

### 問題2: カラム名が異なる

バックアップファイルのスキーマとローカル環境のスキーマが異なる可能性があります。

```powershell
# usersテーブルの構造を確認
docker compose exec db psql -U postgres -d toybox -c "\d users"
```
