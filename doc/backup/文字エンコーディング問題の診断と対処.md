# 文字エンコーディング問題の診断と対処

**作成日**: 2026年1月28日  
**目的**: バックアップファイルの文字エンコーディング問題を診断し、対処する

---

## 現在の状況

`file`コマンドで確認したところ、バックアップファイルが`ASCII text`と表示されています。これは、UTF-8として正しく出力されていない可能性があります。

---

## 詳細な診断

### ステップ1: 実際のINSERT文の内容を確認

```bash
# サーバー側で実行
cd /var/www/toybox/backend

# テスト用のダンプを作成
docker compose exec db pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --column-inserts \
    toybox 2>&1 | head -200 > /tmp/test_dump.sql

# INSERT文のサンプルを確認（文字化けがないか）
echo "=== announcementsテーブルのINSERT文 ==="
grep "INSERT INTO.*announcements" /tmp/test_dump.sql | head -1

echo ""
echo "=== cardsテーブルのINSERT文 ==="
grep "INSERT INTO.*cards" /tmp/test_dump.sql | head -1

echo ""
echo "=== 日本語が含まれるINSERT文を確認 ==="
grep "INSERT INTO" /tmp/test_dump.sql | grep -E "[あ-ん]|[ア-ン]|[一-龯]" | head -3
```

### ステップ2: データベースの実際のエンコーディングを確認

```bash
# サーバー側で実行
echo "=== データベースのエンコーディング ==="
docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
docker compose exec db psql -U postgres -d toybox -c "SHOW client_encoding;"
docker compose exec db psql -U postgres -c "SELECT datname, pg_encoding_to_char(encoding) as encoding FROM pg_database WHERE datname = 'toybox';"

# 実際のデータを確認（文字化けがないか）
echo ""
echo "=== 実際のデータの確認 ==="
docker compose exec db psql -U postgres -d toybox -c "SELECT code, name FROM cards WHERE code = 'C001';"
docker compose exec db psql -U postgres -d toybox -c "SELECT id, title FROM announcements LIMIT 1;"
```

### ステップ3: pg_dumpの出力を直接確認

```bash
# サーバー側で実行
# pg_dumpの出力を直接確認（エンコーディング変換なし）
docker compose exec db pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --column-inserts \
    toybox 2>&1 | grep "INSERT INTO.*cards" | head -1 | od -c | head -20
```

---

## 考えられる原因と対処方法

### 原因1: データベース自体のエンコーディングがUTF8ではない

**確認方法**:
```bash
docker compose exec db psql -U postgres -c "SELECT datname, pg_encoding_to_char(encoding) as encoding FROM pg_database WHERE datname = 'toybox';"
```

**対処方法**: データベースをUTF8で再作成する必要があります（本番環境では慎重に）

### 原因2: pg_dumpの出力が正しくUTF-8として処理されていない

**確認方法**: INSERT文のサンプルを確認して、文字化けが発生しているか確認

**対処方法**: バックアップスクリプトを修正して、明示的にUTF-8として出力する

### 原因3: データベース内のデータ自体が文字化けしている

**確認方法**: サーバー側で直接データを確認

**対処方法**: データベース内のデータを修復する必要があります

---

## 推奨される対処方法

### 方法1: バックアップスクリプトを修正して、明示的にUTF-8として出力

```bash
# サーバー側で実行
# バックアップスクリプトを修正
vi /var/www/toybox/scripts/backup_database.sh
```

以下のように修正：

```bash
# 修正前
docker exec "$CONTAINER_NAME" pg_dump \
    -U "$DB_USER" \
    --encoding=UTF8 \
    ...

# 修正後（環境変数を明示的に設定）
export PGCLIENTENCODING=UTF8
docker exec "$CONTAINER_NAME" pg_dump \
    -U "$DB_USER" \
    --encoding=UTF8 \
    ...
```

### 方法2: データベースのエンコーディングを確認して、必要に応じて再作成

データベースのエンコーディングがUTF8でない場合、UTF8で再作成する必要があります。

**注意**: 本番環境では慎重に実行してください。
