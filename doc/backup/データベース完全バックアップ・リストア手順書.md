## 概要

TOYBOXのデータベースバックアップ・リストアシステムは、以下の要件を満たすように設計されています：

### 重要な原則

> **バックアップは「取得できる」だけでは不十分です。**  
> **「復元できる」ことが確認できて初めて意味があります。**

### 対象データ

- **PostgreSQLデータベース**: ユーザー、投稿、カード、称号などのデータ
- **メディアボリューム**: カード画像、称号画像などのファイル

### 主な課題と解決策

| 課題 | 原因 | 解決策 |
|------|------|--------|
| **文字化け** | エンコーディング設定の不足 | UTF-8を明示的に設定 |
| **画像未復元** | メディアボリュームが別管理 | 画像ボリュームも併せて復元 |
| **データ型エラー** | 列の順序不一致 | `--column-inserts`オプション使用 |

## バックアップの種類と保存先

### サーバー側のバックアップ構成

| 種類 | 保存先 | ファイル名形式 | 頻度 | 保持期間 |
|------|--------|----------------|------|----------|
| **PostgreSQLダンプ** | `/backup/toybox/database/` | `toybox_YYYYMMDD_HHMMSS.sql.gz` | 毎日 3:00 | 7日 |
| **ボリューム完全** | `/backup/toybox/volumes/` | `postgres_data_YYYYMMDD.tar.gz`<br>`media_volume_YYYYMMDD.tar.gz` | 毎週日曜 3:30 | 14日 |
| **ボリューム差分** | `/backup/toybox/volumes-incremental/` | `base_YYYYMMDD/`<br>`incr_YYYYMMDD/` | 毎日 3:45 | 14日 |

### 各方式の特徴

| 方式 | サイズ | 復元速度 | 確実性 | 推奨用途 |
|------|--------|----------|--------|----------|
| PostgreSQLダンプ | 小 | 速い | 高 | **日常的な復元（推奨）** |
| ボリューム完全 | 大 | 中 | 高 | 週次バックアップ、完全復旧 |
| ボリューム差分 | 小 | やや遅い | 中 | 特定日時への復元 |

## 文字化け問題と対策

### 問題の発見方法

データベースから日本語データを取得したときに、以下のように表示される場合は文字化けが発生しています：

```bash
# 正常な場合
name: トラブルリキッド・セイバー
attribute: 水

# 文字化けしている場合
name: ?????????????
attribute: ?
```

### 文字化けの根本原因

データベースに保存されている時点で既に文字化けしている場合：

```bash
# バイト列を確認すると 0x3f（?）になっている
docker exec backend-db-1 psql -U postgres -d toybox -c \
  "SELECT encode(name::bytea, 'hex') FROM cards WHERE id = 14;"
# 結果: 3f3f3f3f3f3f3f... (すべて ? = 0x3f)
```

### 対策の3本柱

#### 1. SQLファイル自体のエンコーディング確認

```powershell
# PowerShellでファイルのエンコーディングを確認
$bytes = [System.IO.File]::ReadAllBytes("toybox_20260129_133225.sql")
$bytes[0..3] | ForEach-Object { $_.ToString("X2") }

# UTF-8として読み込んで日本語が正しく表示されるか確認
Get-Content toybox_20260129_133225.sql -Encoding UTF8 | Select-String "トラブル"
```

#### 2. バックアップ取得時のUTF-8設定

サーバー側のバックアップスクリプトで以下を設定：

```bash
docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 backend-db-1 pg_dump \
    -U postgres \
    --encoding=UTF8 \
    --column-inserts \
    toybox > backup.sql
```

**重要なオプション**：
- `-e PGCLIENTENCODING=UTF8`: PostgreSQLクライアントのエンコーディング
- `-e LANG=C.UTF-8`: システムロケール
- `--encoding=UTF8`: 出力ファイルのエンコーディング
- `--column-inserts`: 列名を明示（順序問題を回避）

#### 3. リストア時のUTF-8設定とコンテナ内実行

**❌ 間違った方法（文字化けする）**：

```powershell
# PowerShellのパイプ経由でデータを渡すと文字化けする
Get-Content backup.sql -Encoding UTF8 | docker exec -T db psql -U postgres toybox
```

**✅ 正しい方法（文字化けしない）**：

```powershell
# 1. ファイルをコンテナ内にコピー
docker cp toybox_20260129_133225.sql backend-db-1:/tmp/restore.sql

# 2. コンテナ内から直接psqlで実行
docker exec backend-db-1 bash -c "PGCLIENTENCODING=UTF8 psql -U postgres -d toybox -f /tmp/restore.sql"

# 3. 確認
docker exec backend-db-1 psql -U postgres -d toybox -c \
  "SELECT id, name, attribute FROM cards WHERE id IN (14, 18);"
```

## バックアップ方法

### サーバー側の自動バックアップ設定

#### ステップ1: バックアップスクリプトの確認

```bash
# サーバーに接続
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144

# バックアップスクリプトの場所を確認
ls -lh /var/www/toybox/scripts/backup_database.sh
```

#### ステップ2: スクリプトの内容確認

バックアップスクリプトには以下の設定が必要です：

```bash
#!/bin/bash
# TOYBOXデータベース自動バックアップスクリプト

set -euo pipefail

BACKUP_DIR="/backup/toybox/database"
DATE=$(date +%Y%m%d_%H%M%S)
CONTAINER_NAME="backend-db-1"
DATABASE_NAME="toybox"
DB_USER="postgres"

# UTF-8エンコーディングを明示的に設定してバックアップ
docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 "$CONTAINER_NAME" pg_dump \
    -U "$DB_USER" \
    --encoding=UTF8 \
    --no-owner \
    --no-privileges \
    --clean \
    --if-exists \
    --verbose \
    --column-inserts \
    "$DATABASE_NAME" | gzip > "$BACKUP_DIR/toybox_${DATE}.sql.gz"
```

**重要なポイント**：
- `--column-inserts`: INSERT文で列名を明示（列順序の問題を回避）
- `--encoding=UTF8`: UTF-8エンコーディング指定
- `-e PGCLIENTENCODING=UTF8`: クライアントエンコーディング設定
- `--no-owner`, `--no-privileges`: 復元時のエラー防止

#### ステップ3: cronで自動実行設定

```bash
# crontabを編集
crontab -e

# 毎日午前3時にバックアップを実行
0 3 * * * /var/www/toybox/scripts/backup_database.sh
```

### 手動バックアップの実行

#### サーバー側で実行

```bash
# サーバーに接続
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144

# 手動でバックアップを実行

# 作成されたバックアップを確認
ls -lh /backup/toybox/database/toybox_*.sql.gz | tail -1
```

#### ローカル環境で実行

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# 現在のデータベースをバックアップ

docker compose exec -T db pg_dump -U postgres toybox > $backupName

Write-Host "バックアップ完了: $backupName" -ForegroundColor Green
```

## リストア方法

### サーバー→サーバー

本番環境でデータベースを復元する手順です。

#### ステップ1: 復元前のバックアップ（重要）

```bash
# サーバーに接続
ssh -i "C:\Users\ayato\.ssh\toybox-2025-11-06-11-40.pem" root@160.251.168.144

# 現在の状態をバックアップ
DATE=$(date +%Y%m%d_%H%M%S)
docker exec backend-db-1 pg_dump -U postgres toybox | gzip > /backup/toybox/database/before_restore_${DATE}.sql.gz

echo "現在の状態をバックアップしました: before_restore_${DATE}.sql.gz"
```

#### ステップ2: 復元するバックアップファイルを選択

```bash
# バックアップファイル一覧を表示
ls -lh /backup/toybox/database/toybox_*.sql.gz

# 復元するファイルを指定（例：最新のファイル）
RESTORE_FILE=$(ls -t /backup/toybox/database/toybox_*.sql.gz | head -1)
echo "復元するファイル: $RESTORE_FILE"
```

#### ステップ3: データベースをリセット

```bash
# データベースを削除して再作成（UTF-8を明示的に指定）
docker exec backend-db-1 psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker exec backend-db-1 psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"

echo "データベースをリセットしました"
```

#### ステップ4: バックアップから復元

```bash
# 解凍しながら復元（UTF-8エンコーディングを設定）
gunzip -c "$RESTORE_FILE" | docker exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -i backend-db-1 psql -U postgres toybox

echo "復元が完了しました"
```

#### ステップ5: 復元結果を確認

```bash
# データベースの接続確認
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT version();"

# データ件数を確認
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT COUNT(*) FROM users;"
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT COUNT(*) FROM submissions;"
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT COUNT(*) FROM cards;"

# 日本語が正しく表示されるか確認
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT id, name FROM cards LIMIT 5;"
```

```powershell
Write-Host "復元が完了しました" -ForegroundColor Green

# ステップC: 一時ファイルを削除（オプション）
docker exec backend-db-1 rm /tmp/restore.sql
```

## トラブルシューティング

### 問題1: 日本語が「?」と表示される（文字化け）

#### 原因の診断

```powershell
# データベースに保存されているデータのバイト列を確認
docker exec backend-db-1 psql -U postgres -d toybox -c `
  "SELECT id, code, encode(name::bytea, 'hex') as name_hex FROM cards WHERE id = 14;"

# 結果が 3f3f3f3f... (すべて 0x3f = ?) の場合、データベースに保存されている時点で既に文字化けしている
```

#### 解決方法A: SQLファイルのエンコーディングを確認

```powershell
# SQLファイルがUTF-8として正しく読めるか確認
Get-Content "toybox_20260129_133225.sql" -Encoding UTF8 | Select-String "トラブル"

# 正しく表示される場合：SQLファイル自体はUTF-8
# 表示されない場合：SQLファイルのエンコーディングが異なる可能性
```

#### 解決方法B: 正しいリストア方法を使用

```powershell
# ファイルをコンテナ内にコピーしてから復元
docker cp toybox_20260129_133225.sql backend-db-1:/tmp/restore.sql
docker exec backend-db-1 bash -c "PGCLIENTENCODING=UTF8 psql -U postgres -d toybox -f /tmp/restore.sql"
```

#### 解決方法C: データベースを再作成

```powershell
# データベースをUTF-8を明示的に指定して再作成
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

### 問題2: 画像が表示されない

#### 確認事項

```powershell
# 1. メディアボリュームのバックアップが存在するか確認
Get-ChildItem C:\backup\toybox-verification\volumes\media_volume_*.tar.gz

# 2. 画像ファイルが存在するか確認
docker compose exec web ls -la /app/public/uploads/cards/
docker compose exec web ls -la /app/public/uploads/titles/

# 3. ボリュームが正しくマウントされているか確認

```

#### 解決方法

メディアボリュームのバックアップをダウンロードして復元してください（[メディアボリューム（画像）の復元](#メディアボリューム画像の復元)を参照）。

### 問題3: 「no configuration file provided: not found」エラー

#### 原因

`docker-compose.yml` が存在しないディレクトリで `docker-compose` コマンドを実行している。

#### 解決方法

```powershell
# 方法A: docker-compose.ymlがあるディレクトリに移動
cd C:\github\toybox\backend
docker compose exec db psql -U postgres -d toybox -c "SELECT 1;"

# 方法B: dockerコマンドを直接使用（移動不要）
docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT 1;"
```

### 問題4: 「FATAL: database "toybox" does not exist」エラー

#### 原因

データベースが存在しない。

#### 解決方法

```powershell
# データベースを作成
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

### 問題5: リストア後にWebアプリケーションが起動しない

#### 原因

データベースのマイグレーションが適用されていない可能性。

#### 解決方法

```powershell
cd C:\github\toybox\backend
docker compose exec web python3 manage.py migrate
```

## まとめ

この手順書に従うことで、以下が実現できます：

### ✅ バックアップ

- UTF-8エンコーディングを明示的に設定
- 列の順序問題を回避（`--column-inserts`）
- 自動バックアップの設定
- バックアップファイルの検証

### ✅ リストア

- サーバー→サーバー：本番環境での復元
- サーバー→ローカル：開発環境での検証
- 文字化けを防ぐ正しい方法
- メディアボリューム（画像）の復元

### ✅ トラブルシューティング

- 文字化け問題の診断と解決
- 画像未復元の解決
- 各種エラーの対処

### 定期的な復元テストの重要性

バックアップは取得するだけでは不十分です。定期的に復元テストを実施することで、以下を確認できます：

- ✅ バックアップファイルが破損していない
- ✅ 復元手順が正しい
- ✅ 復元に必要な時間を把握できる
- ✅ 緊急時にスムーズに対応できる

**推奨**: 月に1回、復元テストを実施してください。
