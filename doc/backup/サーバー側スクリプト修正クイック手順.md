# サーバー側スクリプト修正クイック手順

**目的**: 「cannot execute: required file not found」エラーを修正する

---

## 問題の原因

WinSCPでファイルをコピーした際に、改行コードがWindows形式（CRLF）になっている可能性があります。Linuxでは、改行コードはLF（\n）である必要があります。

---

## 解決方法

### 方法1: 改行コードを変換して修正（推奨）

サーバー側で以下のコマンドを実行してください：

```bash
# 1. 現在のスクリプトの改行コードを確認
head -1 /var/www/toybox/scripts/backup_database.sh | od -c | head -1
# \r が含まれている場合は改行コードを変換する必要があります

# 2. 改行コードをLFに変換
sed -i 's/\r$//' /var/www/toybox/scripts/backup_database.sh

# 3. 実行権限を設定
chmod +x /var/www/toybox/scripts/backup_database.sh

# 4. 改行コードが正しく変換されたか確認
head -1 /var/www/toybox/scripts/backup_database.sh | od -c | head -1
# 出力例: "0000000   #   !   /   b   i   n   /   b   a   s   h  \n"
# \r（キャリッジリターン）が含まれていないことを確認

# 5. スクリプトの構文を確認
bash -n /var/www/toybox/scripts/backup_database.sh
# エラーが表示されなければOK

# 6. スクリプトを実行してテスト
/var/www/toybox/scripts/backup_database.sh
```

### 方法2: スクリプトを再作成（確実）

もし方法1で解決しない場合は、スクリプトを再作成してください：

```bash
# 1. 既存のスクリプトをバックアップ
cp /var/www/toybox/scripts/backup_database.sh /var/www/toybox/scripts/backup_database.sh.backup

# 2. スクリプトを削除
rm /var/www/toybox/scripts/backup_database.sh

# 3. 新しいスクリプトを作成（[バックアップスクリプト修正手順.md](./バックアップスクリプト修正手順.md)の「方法B」を参照）
# または、WinSCPで再度アップロード後、方法1の手順を実行
```

---

## 確認方法

スクリプトが正しく動作するか確認：

```bash
# 1. 実行権限を確認
ls -l /var/www/toybox/scripts/backup_database.sh
# 出力例: "-rwxr-xr-x 1 root root ..." （xが含まれていることを確認）

# 2. シェバン行を確認
head -1 /var/www/toybox/scripts/backup_database.sh
# 出力: "#!/bin/bash"

# 3. 改行コードを確認
head -1 /var/www/toybox/scripts/backup_database.sh | od -c | head -1
# 出力に \r が含まれていないことを確認

# 4. 構文チェック
bash -n /var/www/toybox/scripts/backup_database.sh
# エラーが表示されなければOK

# 5. 実際に実行
/var/www/toybox/scripts/backup_database.sh
```

## トラブルシューティング

### 「sed: cannot rename」エラーが出る場合

ファイルがロックされている可能性があります。別の方法で変換：

```bash
# trコマンドを使用
tr -d '\r' < /var/www/toybox/scripts/backup_database.sh > /tmp/backup_database.sh
mv /tmp/backup_database.sh /var/www/toybox/scripts/backup_database.sh
chmod +x /var/www/toybox/scripts/backup_database.sh
```

### 「bash: /var/www/toybox/scripts/backup_database.sh: /bin/bash^M: bad interpreter」エラーが出る場合

改行コードがまだCRLFのままです。再度変換：

```bash
sed -i 's/\r$//' /var/www/toybox/scripts/backup_database.sh
```

### dos2unixがインストールされている場合

より簡単に変換できます：

```bash
dos2unix /var/www/toybox/scripts/backup_database.sh
chmod +x /var/www/toybox/scripts/backup_database.sh
```

## 次のステップ

スクリプトが正しく実行できるようになったら：

1. 新しいバックアップを取得：
   ```bash
   /var/www/toybox/scripts/backup_database.sh
   ```

2. バックアップファイルを確認：
   ```bash
   ls -lh /backup/toybox/database/toybox_*.sql.gz | tail -1
   ```

3. 新しいバックアップをローカルにダウンロードして、[バックアップ検証手順書](./バックアップ検証手順書.md)に従ってリストアテストを実行
