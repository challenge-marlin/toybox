# リストア実行コマンド（ステップバイステップ）

## ステップ1: バックアップファイルを確認

```powershell
# 最新のSQLバックアップを確認
Get-ChildItem C:\backup\toybox\database\toybox_*.sql.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1
```

ファイルが表示されることを確認してください。

---

## ステップ2: データベースをリセット

```powershell
cd C:\github\toybox\backend
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox WITH ENCODING='UTF8' LC_COLLATE='C' LC_CTYPE='C' TEMPLATE=template0;"
```

---

## ステップ3: バックアップファイルを解凍してリストア

### 方法A: WSL/Git Bashを使用（最も簡単）

```powershell
# WSLがインストールされている場合
wsl gunzip -c "/mnt/c/backup/toybox/database/toybox_20260128_175028.sql.gz" | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```

**注意**: `toybox_20260128_175028.sql.gz` は、ステップ1で確認したファイル名に置き換えてください。

### 方法B: 手動で解凍してからリストア

#### B-1. ファイルを解凍

エクスプローラーで以下を実行：
1. `C:\backup\toybox\database\` を開く
2. `toybox_*.sql.gz` ファイルを右クリック
3. 「7-Zip」→「ここに解凍」を選択
4. `toybox_*.sql` ファイルが作成されることを確認

#### B-2. 解凍したファイルをリストア

```powershell
# ファイル名を確認してから実行（例）
$content = Get-Content "C:\backup\toybox\database\toybox_20260128_175028.sql" -Encoding UTF8 -Raw
$content | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```

**注意**: `toybox_20260128_175028.sql` は、実際のファイル名に置き換えてください。

## ステップ4: メディアボリュームをリストア

```powershell
# 最新のメディアボリュームバックアップを確認
Get-ChildItem C:\backup\toybox\volumes\media_volume_*.tar.gz | Sort-Object LastWriteTime -Descending | Select-Object -First 1

# リストア（ファイル名を置き換えて実行）
docker run --rm -v backend_media_volume:/data -v "C:\backup\toybox\volumes:/backup" alpine sh -c "cd / && tar xzf /backup/media_volume_20260128.tar.gz"
```

**注意**: `media_volume_20260128.tar.gz` は、実際のファイル名に置き換えてください。

## ステップ5: 復元の確認

```powershell
# データの件数を確認
docker compose exec db psql -U postgres -d toybox -c "SELECT COUNT(*) FROM users;"

# 日本語が正しく表示されるか確認
docker compose exec db psql -U postgres -d toybox -c "SELECT id, username, display_name FROM users LIMIT 5;"

# 画像ファイルが存在するか確認
docker compose exec web ls -la /app/public/uploads/cards/ | Select-Object -First 10
```

## トラブルシューティング

### WSLがインストールされていない場合

Git Bashをインストールして使用してください：
1. Git for Windowsをインストール: https://git-scm.com/download/win
2. Git Bashを起動
3. 以下のコマンドを実行：

```bash
gunzip -c /c/backup/toybox/database/toybox_20260128_175028.sql.gz | docker compose exec -e PGCLIENTENCODING=UTF8 -e LANG=C.UTF-8 -T db psql -U postgres toybox
```
