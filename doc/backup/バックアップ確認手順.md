# バックアップ確認手順

**目的**: サーバー側でバックアップが正常に実行され、文字化けが解消されているか確認する

---

## ステップ1: バックアップスクリプトが正しく更新されているか確認

```bash
# サーバー側で実行
# バックアップスクリプトの内容を確認（一時ファイル方式になっているか）
grep -A 5 "一時SQLファイルに書き込み中" /var/www/toybox/scripts/backup_database.sh

# または、pg_dumpの出力先を確認
grep -A 10 "pg_dump" /var/www/toybox/scripts/backup_database.sh | grep -E "(TEMP_SQL_FILE|gzip)"
```

**期待される出力**:
- `log "一時SQLファイルに書き込み中..."`
- `> "$TEMP_SQL_FILE"`
- `gzip -c "$TEMP_SQL_FILE" > "$BACKUP_FILE"`

---

## ステップ2: 最新のバックアップログを確認

```bash
# サーバー側で実行
# 最新のバックアプログを確認
tail -50 /var/log/toybox_backup.log

# エラーがないか確認
grep -i "error\|失敗\|failed" /var/log/toybox_backup.log | tail -10
```

**確認事項**:
- ✅ `一時SQLファイルに書き込み中...`というログがあるか
- ✅ `バックアップファイルを圧縮中...`というログがあるか
- ✅ `✅ Backup successful`というログがあるか
- ❌ エラーメッセージがないか

---

## ステップ3: 最新のバックアップファイルを確認

```bash
# サーバー側で実行
# 最新のバックアップファイルを確認
ls -lth /backup/toybox/database/toybox_*.sql.gz | head -1

# ファイルサイズを確認（0バイトでないことを確認）
ls -lh /backup/toybox/database/toybox_*.sql.gz | tail -1
```

**確認事項**:
- ✅ ファイルが存在する
- ✅ ファイルサイズが0バイトでない（通常は数MB以上）

---

## ステップ4: バックアップファイルの内容を確認（文字化けチェック）

```bash
# サーバー側で実行
# 最新のバックアップファイル名を取得
LATEST_BACKUP=$(ls -t /backup/toybox/database/toybox_*.sql.gz | head -1)
echo "最新のバックアップファイル: $LATEST_BACKUP"

# バックアップファイルを解凍して内容を確認
gunzip -c "$LATEST_BACKUP" | grep "INSERT INTO public.titles" | head -1

# cardsテーブルのINSERT文も確認
gunzip -c "$LATEST_BACKUP" | grep "INSERT INTO public.cards" | head -1 | head -c 200
```

**期待される結果**:
- ✅ 日本語が正しく表示される（`蒸気の旅人`、`真鍮の探究者`など）
- ❌ `?????`や`���C���l`のような文字化けが発生していない

**文字化けが発生している場合**:
- バックアップスクリプトが正しく更新されていない可能性があります
- ステップ1に戻って、スクリプトの内容を確認してください

---

## ステップ5: 一時SQLファイルの確認（デバッグ用）

一時SQLファイルは通常、バックアップ完了後に削除されます。確認したい場合は、バックアップスクリプトを一時的に修正して一時ファイルを残すことができます：

```bash
# サーバー側で実行
# バックアップスクリプトをバックアップ
cp /var/www/toybox/scripts/backup_database.sh /var/www/toybox/scripts/backup_database.sh.backup

# 一時ファイルを削除する行をコメントアウト（デバッグ用）
sed -i 's/rm -f "\$TEMP_SQL_FILE"/# rm -f "$TEMP_SQL_FILE"  # デバッグ用: 一時的にコメントアウト/' /var/www/toybox/scripts/backup_database.sh

# バックアップを実行
/var/www/toybox/scripts/backup_database.sh

# 一時SQLファイルを確認
ls -lh /backup/toybox/database/toybox_*.sql | tail -1

# 一時SQLファイルの内容を確認（日本語が正しく含まれているか）
LATEST_SQL=$(ls -t /backup/toybox/database/toybox_*.sql | head -1)
grep "INSERT INTO public.titles" "$LATEST_SQL" | head -1

# デバッグが終わったら、元に戻す
cp /var/www/toybox/scripts/backup_database.sh.backup /var/www/toybox/scripts/backup_database.sh
```

## トラブルシューティング

### 問題1: 一時SQLファイルが表示されない

**原因**: バックアップが正常に完了し、一時ファイルが削除されたため

**確認方法**:
1. バックアップログを確認（ステップ2）
2. 最新のバックアップファイルが存在するか確認（ステップ3）
3. バックアップファイルの内容を確認（ステップ4）

### 問題2: バックアップファイルが作成されない

**確認事項**:
1. バックアップスクリプトに実行権限があるか: `ls -l /var/www/toybox/scripts/backup_database.sh`
2. Dockerコンテナが起動しているか: `docker ps | grep backend-db-1`
3. PostgreSQLに接続できるか: `docker exec backend-db-1 pg_isready -U postgres`
4. バックアップディレクトリの権限: `ls -ld /backup/toybox/database`

### 問題3: 文字化けが続く

**確認事項**:
1. バックアップスクリプトが正しく更新されているか（ステップ1）
2. サーバー側のデータベース内のデータが正しいか:
   ```bash
   docker exec backend-db-1 psql -U postgres -d toybox -c "SELECT id, name FROM titles LIMIT 3;"
   ```
3. サーバー側のシェル環境のエンコーディング:
   ```bash
   echo $LANG
   locale
   ```
