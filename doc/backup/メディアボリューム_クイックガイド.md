# メディアボリューム バックアップ・リストア クイックガイド

## 🎯 概要

本番環境のメディアファイル（画像など）をバックアップし、ローカル環境でリストアする手順。

## ⚡ クイック実行手順

### ステップ1: 本番環境でバックアップ（サーバー側）

```bash
# 本番サーバーにSSH接続
ssh user@production-server

# バックアップスクリプトをアップロード（初回のみ）
# scp scripts/backup_media_from_prod.sh user@production-server:/tmp/

# スクリプトに実行権限を付与
chmod +x /tmp/backup_media_from_prod.sh

# バックアップ実行
sudo /tmp/backup_media_from_prod.sh

# 実行結果を確認
# - バックアップファイル: /backup/toybox/media/media_volume_YYYYMMDD_HHMMSS.tar.gz
# - 情報ファイル: /backup/toybox/media/media_backup_YYYYMMDD_HHMMSS_info.txt
# - ハッシュファイル: /backup/toybox/media/media_volume_YYYYMMDD_HHMMSS.tar.gz.sha256
```

### ステップ2: ローカルにダウンロード（Windowsローカル）

```powershell
# PowerShellで実行
cd C:\github\toybox\backend\media_backups

# バックアップファイルをダウンロード（タイムスタンプを実際の値に置き換え）
$DATE = "20260129_XXXXXX"  # バックアップのタイムスタンプ

scp user@production-server:/backup/toybox/media/media_volume_${DATE}.tar.gz ./
scp user@production-server:/backup/toybox/media/media_volume_${DATE}.tar.gz.sha256 ./
scp user@production-server:/backup/toybox/media/media_backup_${DATE}_info.txt ./
```

### ステップ3: ローカルでリストア（Windowsローカル）

```powershell
# PowerShellで実行
cd C:\github\toybox\backend

# リストア実行（タイムスタンプを実際の値に置き換え）
.\restore_media_local.ps1 -BackupFile "media_volume_20260129_XXXXXX.tar.gz"

# 確認プロンプトで "yes" を入力
```

### ステップ4: 検証（Windowsローカル）

```powershell
# Dockerコンテナを起動
cd C:\github\toybox\backend
docker compose up -d

# 検証スクリプトを実行
.\verify_media_restore.ps1

# より詳細な検証
.\verify_media_restore.ps1 -Detailed
```

### ステップ5: ブラウザで確認

1. http://localhost:8000 にアクセス
2. ログインしてプロファイル画像が表示されるか確認
3. ガチャページでカード画像が表示されるか確認
4. 開発者ツール（F12）で404エラーがないか確認

## 🔧 スクリプトオプション

### restore_media_local.ps1 のオプション

```powershell
# 基本実行
.\restore_media_local.ps1 -BackupFile "media_volume_20260129_120000.tar.gz"

# 確認なしで実行（自動化用）
.\restore_media_local.ps1 -BackupFile "media_volume_20260129_120000.tar.gz" -Force

# 既存ボリュームのバックアップをスキップ
.\restore_media_local.ps1 -BackupFile "media_volume_20260129_120000.tar.gz" -SkipBackup

# 両方のオプションを使用
.\restore_media_local.ps1 -BackupFile "media_volume_20260129_120000.tar.gz" -Force -SkipBackup
```

### verify_media_restore.ps1 のオプション

```powershell
# 基本検証
.\verify_media_restore.ps1

# 詳細検証（全ファイルリストを表示）
.\verify_media_restore.ps1 -Detailed
```

## ⚠️ トラブルシューティング

### ハッシュ値が一致しない

```powershell
# ファイルを再ダウンロード
scp user@production-server:/backup/toybox/media/media_volume_${DATE}.tar.gz ./

# それでも一致しない場合は強制実行
.\restore_media_local.ps1 -BackupFile "media_volume_${DATE}.tar.gz" -Force
```

### Dockerコンテナが起動しない

```powershell
# コンテナのログを確認
docker compose logs web

# コンテナを再起動
docker compose down
docker compose up -d
```

### 画像が表示されない

```powershell
# ファイルパーミッションを修正
docker run --rm -v backend_media_volume:/data alpine chmod -R 755 /data

# Webコンテナを再起動
docker compose restart web

# ブラウザのキャッシュをクリア
# Ctrl+Shift+R または Ctrl+F5
```

### メディアボリュームをクリアしたい

```powershell
# すべてのメディアファイルを削除
docker run --rm -v backend_media_volume:/data alpine sh -c "rm -rf /data/*"

# ボリュームごと削除（コンテナを停止してから）
docker compose down
docker volume rm backend_media_volume
docker compose up -d
```
