# ローカルリストア手順（最新版）

**目的**: サーバーから取得した最新のバックアップファイルをローカル環境にリストアする

---

## 前提条件

- Docker Desktopが起動している
- `backend`ディレクトリに移動済み
- バックアップファイルが`C:\backup\toybox-verification\database\`に配置されている

---

## ステップ1: データベースをリセット

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# データベースをリセット
Write-Host "データベースをリセット中..." -ForegroundColor Yellow
docker compose exec db psql -U postgres -c "DROP DATABASE IF EXISTS toybox;"
docker compose exec db psql -U postgres -c "CREATE DATABASE toybox;"
```

---

## ステップ2: バックアップファイルからリストア

```powershell
# バックアップファイルのパスを指定（実際のファイル名に置き換えてください）
$BACKUP_FILE = "C:\backup\toybox-verification\database\toybox_20260128_175028.sql.gz"
$tempFile = $BACKUP_FILE -replace "\.gz$", ""
$logFile = "$tempFile.log"

# ファイルの存在確認
if (-not (Test-Path $BACKUP_FILE)) {
    Write-Host "エラー: バックアップファイルが見つかりません: $BACKUP_FILE" -ForegroundColor Red
    exit 1
}

Write-Host "`nバックアップファイルを解凍中..." -ForegroundColor Yellow
& "C:\Program Files\7-Zip\7z.exe" e $BACKUP_FILE "-o$(Split-Path $tempFile)" -y | Out-Null

if (-not (Test-Path $tempFile)) {
    Write-Host "エラー: 解凍に失敗しました" -ForegroundColor Red
    exit 1
}

Write-Host "データベースにリストア中（UTF-8で読み込み）..." -ForegroundColor Yellow

# UTF-8を明示的に指定して読み込む（重要！）
try {
    $content = [System.IO.File]::ReadAllText($tempFile, [System.Text.Encoding]::UTF8)
    $content | docker compose exec -T db psql -U postgres toybox 2>&1 | Tee-Object -FilePath $logFile
    
    Write-Host "`nリストアが完了しました" -ForegroundColor Green
} catch {
    Write-Host "エラー: $_" -ForegroundColor Red
    exit 1
}
```

---

## ステップ3: リストア結果の確認

```powershell
# リストア結果を確認
Write-Host "`n=== リストア結果の確認 ===" -ForegroundColor Cyan

# エラーを確認（UTF-8で読み込む）
$errors = Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" } | Measure-Object
Write-Host "エラー数（already exists以外）: $($errors.Count)件" -ForegroundColor $(if ($errors.Count -eq 0) { "Green" } else { "Yellow" })

if ($errors.Count -gt 0) {
    Write-Host "`nエラー内容（最初の10件）:" -ForegroundColor Yellow
    Get-Content $logFile -Encoding UTF8 | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" } | Select-Object -First 10
}

# データ件数を確認
Write-Host "`n=== データ件数の確認 ===" -ForegroundColor Cyan
docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
UNION ALL
SELECT 'announcements', COUNT(*) FROM announcements
UNION ALL
SELECT 'titles', COUNT(*) FROM titles
UNION ALL
SELECT 'user_meta', COUNT(*) FROM user_meta
ORDER BY table_name;
"@
```

## ステップ4: 日本語データの確認

```powershell
# 日本語が正しく表示されるか確認
Write-Host "`n=== 日本語データの確認 ===" -ForegroundColor Cyan

Write-Host "`n【カードデータ（最初の5件）】" -ForegroundColor Yellow
docker compose exec db psql -U postgres -d toybox -c "SET client_encoding TO 'UTF8'; SELECT id, code, name FROM cards LIMIT 5;"

Write-Host "`n【称号データ】" -ForegroundColor Yellow
docker compose exec db psql -U postgres -d toybox -c "SET client_encoding TO 'UTF8'; SELECT id, name FROM titles ORDER BY id;"
```

**注意**: PowerShellのコンソール出力で日本語が`?`と表示される場合がありますが、これはコンソール表示の問題で、データベース内のデータ自体は正しい可能性があります。管理画面で確認してください。

## トラブルシューティング

### 問題1: 「cannot execute: required file not found」エラー

**原因**: バックアップファイルが存在しない、またはパスが間違っている

**解決方法**:
```powershell
# バックアップファイルの存在を確認
Test-Path "C:\backup\toybox-verification\database\toybox_20260128_175028.sql.gz"

# ファイル一覧を確認
Get-ChildItem "C:\backup\toybox-verification\database\*.sql.gz" | Sort-Object LastWriteTime -Descending | Select-Object -First 5
```

### 問題2: 文字化けが発生する

**原因**: PowerShellのコンソール出力の問題、またはデータベース内のデータ自体が文字化けしている

**確認方法**:
1. 管理画面で確認（最も確実）
2. データベース内のデータを直接確認:
   ```powershell
   docker compose exec db psql -U postgres -d toybox -c "SET client_encoding TO 'UTF8'; SELECT name FROM titles LIMIT 1;"
   ```

### 問題3: `titles`テーブルでユニークインデックスエラー

**原因**: サーバー側のデータベースに既に重複データが存在している

**対処方法**:
- このエラーはインデックスの作成に失敗しただけで、データ自体はリストアされています
- サーバー側のデータベースで重複データをクリーンアップする必要があります
