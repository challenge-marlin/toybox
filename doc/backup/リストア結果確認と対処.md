# リストア結果確認と対処

**作成日**: 2026年1月28日  
**目的**: リストア結果を確認し、問題があれば対処する

---

## 現在の状況

リストア時に以下のエラーが発生していますが、多くのINSERT文は成功しています：

- ✅ `INSERT 0 1`が多数表示されている → データはリストアされている
- ❌ 文字エンコーディングの問題（文字化け）が一部で発生
- ❌ 一部のINSERT文で構文エラーが発生

---

## ステップ1: データ件数の確認

まず、どの程度のデータがリストアされたかを確認してください：

```powershell
# backendディレクトリに移動
cd C:\github\toybox\backend

# 各テーブルのデータ件数を確認
Write-Host "=== データ件数の確認 ===" -ForegroundColor Cyan

docker compose exec db psql -U postgres -d toybox -c @"
SELECT 
    'users' as table_name, COUNT(*) as count FROM users
UNION ALL
SELECT 'cards', COUNT(*) FROM cards
UNION ALL
SELECT 'submissions', COUNT(*) FROM submissions
UNION ALL
SELECT 'user_cards', COUNT(*) FROM user_cards
UNION ALL
SELECT 'announcements', COUNT(*) FROM announcements
UNION ALL
SELECT 'titles', COUNT(*) FROM titles
UNION ALL
SELECT 'user_meta', COUNT(*) FROM user_meta
UNION ALL
SELECT 'reactions', COUNT(*) FROM reactions
ORDER BY table_name;
"@
```

## ステップ2: エラーの詳細を確認

```powershell
# ログファイルのパス（例）
$logFile = "C:\backup\toybox-verification\database\toybox_20260128_150118.sql.log"

# エラーを確認
Write-Host "`n=== エラーの詳細 ===" -ForegroundColor Yellow
$errors = Get-Content $logFile | Select-String -Pattern "ERROR:" | Where-Object { $_ -notmatch "already exists" }
$errors | Select-Object -First 20

# エラーの種類を分類
Write-Host "`n=== エラーの種類 ===" -ForegroundColor Yellow
$syntaxErrors = $errors | Select-String -Pattern "syntax error"
$encodingErrors = $errors | Select-String -Pattern "\?"
Write-Host "構文エラー: $($syntaxErrors.Count)件" -ForegroundColor $(if ($syntaxErrors.Count -eq 0) { "Green" } else { "Red" })
Write-Host "文字エンコーディングエラー（推定）: $($encodingErrors.Count)件" -ForegroundColor $(if ($encodingErrors.Count -eq 0) { "Green" } else { "Yellow" })
```

## ステップ3: 成功したINSERT文の数を確認

```powershell
# 成功したINSERT文の数を確認
$successInserts = Get-Content $logFile | Select-String -Pattern "INSERT 0 \d+" | Measure-Object
Write-Host "`n成功したINSERT文: $($successInserts.Count)件" -ForegroundColor Green
```

## 対処方法

### 状況A: 主要テーブルにデータが存在する場合

**例**: `users: 12件`, `cards: 50件以上`, `submissions: 20件以上`

→ **この場合、バックアップは基本的に成功しています**。文字エンコーディングの問題は一部のレコードにのみ影響している可能性があります。

**確認事項**:
1. Webアプリケーションが正常に動作するか確認
2. データが正しく表示されるか確認
3. 文字化けしているレコードがどのテーブルか確認

### 状況B: 主要テーブルが0件の場合

→ バックアップファイル自体に問題がある可能性があります。

**対処方法**:
1. サーバー側のデータベースの文字エンコーディングを確認
2. サーバー側で新しいバックアップを作成

---

## サーバー側のデータベース文字エンコーディング確認

```bash
# サーバー側で実行
cd /var/www/toybox/backend

# データベースの文字エンコーディングを確認
docker compose exec db psql -U postgres -d toybox -c "SHOW server_encoding;"
docker compose exec db psql -U postgres -d toybox -c "SHOW client_encoding;"

# データベースの作成時のエンコーディングを確認
docker compose exec db psql -U postgres -c "SELECT datname, pg_encoding_to_char(encoding) FROM pg_database WHERE datname = 'toybox';"
```

**期待される結果**: `UTF8`
